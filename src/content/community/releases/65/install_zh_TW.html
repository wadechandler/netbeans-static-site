NetBeans IDE 6.5 安裝步驟

上次更新： August 9, 2012
翻譯者: David Lin (社群成員)

本文件說明如何安裝 NetBeans 6.5 到您的系統上。更多有關於支援的作業系統與硬體需求，請參見 NetBeans 6.5 發行說明。參見 NetBeans IDE 6.5 資訊 學習有關 NetBeans IDE 6.5 的新特點。

軟體需求

在您安裝 IDE 之前，必須先安裝 Java SE Development Kit (JDK) 5 Update 16 (版本 1.5.0_16) 或更新 (包含 JDK 6 Update 7) 。假如您沒有安裝 JDK 5 Update 16 或更新的版本，您將無法進行安裝程式。更多有關於 JDK 需求的資訊，請參見軟體需求章節於 NetBeans IDE 6.5 發行說明。

有關 PHP, Ruby 以及 C++ 等下載選項，您可以選擇只安裝 JRE 即可而不需 JDK。然而，假如您想要使用任何有關 Java 方面的功能，就必須安裝 JDK。

頂端

安裝程式下載選項

在 NetBeans IDE 6.5 下載頁面，您可以在數個安裝程式中擇一下載，每個都包含了 base DE 以及額外的工具與功能。

我們提供了下列幾個安裝程式：

  • Java. 提供給 Java SE, Java EE 與 Java ME 應用程式的開發工具。這個下載選項也包含了 GlassFish V2 UR2 應用程式伺服器，GlassFish v3 Prelude 應用程式伺服器與 Apache Tomcat。
  • Java SE. 提供了開發 Java SE 所需的所有標準特性與功能。也包含了支援開發 NetBeans Plugin 等功能。
  • Ruby. 提供了 Ruby 開發工具以及對 Rails and JRuby 的支援。也包含了 GlassFish v3 Prelude 應用程式伺服器。
  • PHP. 提供支援 PHP 開發的工具。PHP 下載選項可在 Early Access 中找到。
  • C/C++. 支援 C/C++ 語言的開發。
  • 全部. 這是 NetBeans IDE 6.5 完全下載選項。它包含了 IDE 的所有功能與套件。包含了開發 Service Oriented Architecture 的工具。

頂端

客製化您的安裝程式

假如您選擇 Java 或是 "全部" 下載選項，您可以在安裝的時候選擇要安裝哪些工具與特性。在歡迎頁面點擊客製化並且選擇您所想要使用的特性。更多資訊，請參見 軟體安裝。

假如您想要增加新功能到您的 NetBeans IDE 中，可以使用 NetBeans 模組管理器。舉例來說，假設您一開始選擇下載與安裝 Ruby 套件。然後您決定您想要安裝 C/C++ 的功能。您就可以到 NetBeans 的模組管理器中，在 IDE 裡面 (選擇工具->模組) 然後增加 C/C++ 套件到您已安裝的 IDE 之中。

我們在此提供了數個對於安裝或設定 NetBeans IDE 特性的幾個重要提示：

  1. Apache Tomcat 6.0.18 包含在 "Java" 以及 "全部" 下載選項中，但是在預設情況下是沒有被安裝的。如果要從 Java 或是 "全部" 下載選項中安裝 Apache Tomcat，請執行安裝程式並且在客製化安裝對話框中選擇 Apache Tomcat 6.0.18。
  2. 至於 C/C++ 開發 (適用於 C/C++ 以及 "全部" 等下載)，您必須額外安裝編譯器和工具。更多資訊，請參見 安裝與設定 C/C++ 支援。
  3. 假如您想要使用 Sun Java System Access Manager, 除了選擇 "全部" 下載選項之外，您還必須安裝 Java Applications Platform SDK Update 5 或是更新版本，它們可從這裡下載 http://java.sun.com/javaee/downloads/index.jsp。
  4. 如欲開發在 CDC 裝置上的 Java ME 應用程式，您必須下載以及安裝 Sun Java Toolkit 1.0 for CDC。在安裝之後，您必須在 IDE 中註冊 CDC 平台。
  5. 對於 Mac OS X 的使用者，您可以從 Java，Ruby，以及 "全部" 等下載，來進行客製化 NetBeans IDE 的安裝。至於 JavaSE, PHP 以及 C/C++ 等下載，IDE 的安裝則不能被客製化。

頂端

開始下載

  1. 前往 NetBeans IDE 6.5 下載頁面。
  2. 在頁面的右上角，請從下拉式選單來選擇您的平台。
  3. 然後從下載頁面，選擇想要安裝的選項，並點擊下載按鈕。
  4. 最後儲存安裝程式到您的系統中。

頂端

軟體安裝

請依照在本章節所說明的指令來將 IDE 安裝在您的系統上。這些安裝指令適用於全部被支援的平台。詳細支援的平台，請參見 NetBeans IDE 6.5 發行說明。

Microsoft Windows, Solaris OS 與 Linux

安裝軟體：

  1. 在下載完成之後，執行安裝程式。
    • 對 Windows 使用者而言，安裝程式的附檔名是 .exe。 請點兩下執行安裝程式。
    • 對於 Solaris 與 Linux 使用者，安裝程式的附檔名是 .sh。在這些平台上，您必須設定安裝程式檔案的權限為可執行檔，可使用下面的命令：
      chmod +x ./<安裝程式檔案名稱>
  2. 選擇工具與特性安裝，請依照下面的步驟在安裝程式精靈的歡迎畫面下操作：
    1. 點擊客製化。
    2. 在客製化安裝對話框中，選擇您所想要安裝的工具與特性。
    3. 點擊 OK 。
  3. 在安裝程式精靈的歡迎畫面，選擇下一步。
  4. 在授權協議同意畫面，閱讀授權協議，選擇同意的選取方塊，然後點擊下一步。
  5. 在 NetBeans IDE 安裝畫面，請依照下列步驟：
    1. 您可以使用預設 NetBeans IDE 的安裝目錄，或是選擇另一個目錄。
      附註: 安裝的目錄必須是空的，並且您用來執行安裝程式的使用者檔案必須要對該目錄有讀取/寫入的權限。
    2. 您可以使用預設的 JDK 來使用 NetBeans IDE 或者選擇從下拉式選單選擇其他的選項。
      假如安裝程式精靈沒有找到合適的 JDK 給 NetBeans IDE 使用，則您的 JDK 不是安裝在預設的路徑。如果是在這種情況下，請將路徑指定到您已安裝的 JDK 的位置，或是取消目前的安裝，然後安裝所需的 JDK 版本，並重新執行安裝程式。
    3. 點擊下一步。
  6. 當進行到 GlassFish V2 UR2 應用程式伺服器的安裝畫面時，請依照下列步驟：
    1. 您可以使用預設的安裝目錄，或是選擇另一個目錄。
      附註: 安裝的目錄必須是空的，並且您用來執行安裝程式的使用者檔案必須要對該目錄有讀取/寫入的權限。
    2. 您可從可用的 JDK 安裝列表中，選擇您想要使用的 JDK 來操作應用程式伺服器，或是點擊瀏覽按鈕來指定其他的 JDK 安裝路徑。
    3. 更改預設的伺服器網域使用者名稱與密碼或是接受預設的值，然後點擊下一步。
      預設的使用者名稱和密碼是 admin 與 adminadmin。
    4. 再次確認伺服器預設的連接埠值 (HTTP, HTTPS 與 Admin) 或是視需要改變他們。
    5. 點擊下一步。
  7. 當 GlassFish v3 Prelude 應用程式伺服器的安裝畫面顯示時，您可以接受預設的安裝目錄或是指定其他的安裝目錄。
  8. 當 Apache Tomcat 的安裝畫面顯示時，您可以接受預設的安裝目錄或是指定其他的安裝目錄。
  9. 在摘要的畫面，請再次確認要被安裝的項目列表是否正確，並且確認您的硬碟有足夠的空間來完成安裝。
  10. 點擊安裝來開始安裝軟體。
  11. 當安裝完成後：
    1. 假如 GlassFish 應用程式伺服器有被安裝，您可選擇當完成安裝程式後註冊 Sun Java System 應用程式伺服器與 NetBeans IDE。
    2. 假如 GlassFish 應用程式伺服器沒有被安裝, 您可選擇當完成安裝程式後註冊 NetBeans IDE。
    3. 點擊完成離開精靈。
  12. 假如您選擇要註冊，將會在瀏覽器開啟註冊畫面。點擊註冊。
  13. 輸入您的 Sun 線上帳號或是到 www.sun.com 來建立一個新的帳號，您可從連結直接連到該頁面。

附註: 假如您在完成整個安裝軟體的時候遇到問題，請參見 疑難排解，該章節提供了關於在安裝的時候可能會遇到的種種問題與解決方案。

Note: 假如您在安裝完成後尚未註冊 GlassFish 應用程式伺服器，您可以在開啟 NetBeans IDE 後，執行應用 GlassFish 程式伺服器，並點擊滑鼠右鍵選擇 Admin console 來進行註冊。

附註: 假如您在安裝完成後尚未註冊 NetBeans IDE，您可以再執行 NetBeans IDE 時註冊。

Mac OS X

安裝軟體：

  1. 當下載完成，請執行安裝程式。安裝程式檔案的附檔名是 .dmg。
  2. 點擊套件圖示。該套件的附檔名是 .mpkg。 安裝精靈啟動。
  3. 在安裝精靈的歡迎畫面，點擊下一步。
  4. 閱讀授權協議，並且點擊下一步。在彈出視窗中選擇同意來接受授權協議。
  5. 在選擇安裝目的地的畫面中，請選擇硬碟並且點擊下一步。
  6. 假如您下載 "全部", Java 或是 Ruby 等安裝選項，您可以客製化您的安裝程式。在安裝精靈的最後一個畫面，按下左手邊的客製化按鈕。功能與特性的樹狀圖會被顯示出來。
  7. 選擇您所想要安裝的功能與特性來安裝。
  8. 輸入您系統的管理者姓名與密碼並點擊 OK 進行安裝。

移除軟體

Microsoft Windows

假如有需要，您可以移除 NetBeans IDE，GlassFish V2 UR2 應用程式伺服器，GlassFish v3 Prelude 應用程式伺服器與 Apache Tomcat。請依照下面步驟:

移除 IDE：

  1. 請關閉 IDE。
  2. 在控制台，選擇新增/移除程式，然後選擇 NetBeans 6.5 並選擇移除。

移除 GlassFish V2 UR2 應用程式伺服器:

  1. 請關閉 IDE。
  2. 在控制台，選擇新增/移除程式，然後選擇 GlassFish V2 UR2 並選擇移除。

移除 GlassFish v3 Prelude 應用程式伺服器：

  1. 請關閉 IDE。
  2. 在控制台，選擇新增/移除程式，然後選擇 GlassFish v3 Prelude 並選擇移除。

移除 Apache Tomcat：

  1. 請關閉 IDE。
  2. 在控制台，選擇新增/移除程式，然後選擇 Apache Tomcat 並選擇移除。

Solaris OS, Linux

假如有需要，您可以移除 NetBeans IDE，GlassFish V2 UR2 應用程式伺服器，GlassFish v3 Prelude 應用程式伺服器與 Apache Tomcat。請依照下面步驟:

移除 IDE：

  1. 請關閉 IDE。
  2. 在 IDE 的安裝目錄，執行 uninstall.sh 檔案。
  3. 如果要一併移除 GlassFish/Tomcat，請確定在摘要畫面選擇它們。
  4. 在摘要畫面，點擊移除。
  5. 當移除完成，點擊完成。

移除 GlassFish V2 UR2 或 V3 Prelude 應用程式伺服器：

  1. 請關閉 IDE。
  2. 請至應用程式伺服器的安裝目錄。

    附註: 預設的安裝目錄可能會視安裝的型態與平台而有所不同。以下的表格顯示預設的應用程式伺服器的安裝目錄。

    平台 安裝成...

    GlassFish V2 UR2

    應用程式伺服器安裝

    		 GlassFish v3 Prelude 應用程式伺服器安裝
    Solaris OS 
    		 root /opt/glassfish-v2ur2 /opt/glassfish-v3-prelude
    Solaris OS,
    Linux    		 user ~/glassfish-v2ur2 ~/glassfish-v3-prelude
    Linux root /usr/local/glassfish-v2ur2 /usr/local/glassfish-v3-prelude
  3. 在應用程式伺服器安裝目錄，請執行 uninstall.sh 檔案。
  4. 在摘要畫面，點擊移除。
  5. 當移除完成，點擊完成。

附註: GlassFish 應用程式伺服器為開發 SOA 功能的必要條件。當您選擇移除應用程式伺服器，SOA 功能將會自動從 NetBeans IDE 中移除。

移除 Apache Tomcat：

  1. 請關閉 IDE。
  2. 請至 Apache Tomcat 安裝目錄並執行 uninstall.sh 檔案。
  3. 在摘要畫面，點擊移除。
  4. 當移除完成，點擊完成。

Mac OS X

移除 IDE 與應用程式伺服器：

  1. 請關閉 IDE。
  2. 直接拖拉程式的資料夾到資源回收桶。
    請注意，預設的 IDE 與應用程式伺服器安裝路徑為 /Applications/NetBeans/。

升級與和先前的版本並存

NetBeans IDE 5.5 或 5.5.1, 6.0, 或 6.1 可以在同一個系統上與 NetBeans IDE 6.5 並存。您不需要為了安裝或執行 NetBeans 6.5 而移除 5.5 或 5.5.1, 6.0, 或 6.1 等版本。

假如您已安裝了 NetBeans IDE 5.5, 5.5.1, 6.0, 或 6.1，您將可以在第一次執行的時候，選擇是否導入先前的使用者設定到新的 NetBeans 6.5 之中。

疑難排解

有關安裝程式的工作目錄

安裝程式在其工作目錄儲存安裝的相關資訊。舉例來說，這些資料可能是被安裝在您系統上的項目列表，安裝時的紀錄訊息，以及註冊值等資料。

Windows 上的安裝程式預設的工作目錄在 %USERPROFILE%\.nbi 而其他平台上則為 ~/.nbi。安裝時的紀錄訊息儲存在 log 子目錄下。

未完成安裝後的清理動作。

當在未完成安裝或是安裝失敗後，您可能會需要從您的系統清理安裝的紀錄。

對於 Microsoft Windows, Solaris OS 與 Linux 的使用者：

  1. 移除安裝程式工作目錄。
  2. 移除 GlassFish 應用程式伺服器與 IDE 的安裝目錄。
  3. 在 Windows 平台上，從下面的登入鍵值移除所有 nbi* 機碼值：
    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall
    HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Mac OS X 的使用者：

  • 拖曳程式的安裝資料夾或檔案到資源回收桶。

在執行安裝之前禁用一些軟體執行。

在執行安裝之前，最好是能先停止防火牆，防毒軟體，與一些監視的應用程式。因為這類的應用程式有可能會防止 IDE 元件在安裝時被正確的設定。

舉例來說，假如您有 Logitech Process Monitor 應用程式 (LVPrcSrv.exe) 在您的系統上執行，請先在安裝前關閉它。
