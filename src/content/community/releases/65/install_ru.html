Указания по установке среды IDE NetBeans 6.5

Дата последнего обновления: August 9, 2012

В этом документе описывается процедура установки среды IDE NetBeans 6.5. Для получения информации о поддерживаемых операционных системах и комплектации оборудования для среды IDE см. Примечания к выпуску среды IDE NetBeans 6.5. Для получения информации о новых возможностях среды IDE NetBeans 6.5 см. Информация по среде IDE NetBeans 6.5.

Требуемое программное обеспечение

Перед установкой среды IDE необходимо установить комплект Java SE Development Kit (JDK) 5 с обновлением 16 (версия 1.5.0_16) или выше (включая комплект JDK 6 с обновлением 7). Если не установлен комплект JDK 5 с обновлением 16 или выше, продолжить установку будет невозможно. Для получения дополнительной информации о требуемой версии комплекта JDK см. "Требуемое программное обеспечение" в документе Примечания к выпуску среды IDE NetBeans 6.5.

Для вариантов загрузки PHP, Ruby и C++ вместо комплекта JDK можно установить среду JRE. Однако если планируется использовать любые функции Java, будет нужен комплект JDK.

в начало

Варианты загрузки программы установки

На странице загрузки среды IDE NetBeans 6.5 можно загрузить одну из нескольких программ установки, каждая из которых содержит базовую среду IDE и дополнительные средства.

Доступны следующие программы установки:

  • Java. Предоставляет средства для разработки приложений Java SE, Java EE и Java ME. Кроме того, данный вариант загрузки включает серверы приложений GlassFish V2 UR2, GlassFish v3 Prelude и пакет Apache Tomcat.
  • Java SE. Предоставляет все стандартные компоненты для разработки Java SE. Кроме того, включена поддержка разработки подключаемых модулей для среды NetBeans.
  • Ruby. Предоставляет средства для разработки приложений Ruby и включает поддержку Rails и JRuby. Также содержит сервер приложений GlassFish v3 Prelude.
  • PHP. Предоставляет средства для разработки PHP. Вариант загрузки "PHP" доступен в режиме "Early Access".
  • C/C++. Поддерживает разработку приложений на языке C/C++.
  • All. Это наиболее полный вариант загрузки для среды IDE NetBeans 6.5. Он содержит все среды выполнения и пакеты, доступные для среды IDE. Включает средства для разработки архитектуры, ориентированной на службы.

в начало

Индивидуализация установки

При выборе вариантов загрузки "Java" или "All" можно точно указать средства и среды выполнения, которые требуется установить. На странице приветствия нажмите кнопку "Customize" и выберите функции, которые нужно использовать. Для получения дополнительной информации см. раздел Установка программного обеспечения.

Для добавления функциональных возможностей к среде IDE NetBeans используется диспетчер подключаемых модулей NetBeans. Например, предположим, что сначала был загружен и установлен пакет "Ruby". После этого было принято решение попробовать функциональность C/C++. Для этого в среде IDE перейдите к диспетчеру подключаемых модулей NetBeans (выберите "Tools > Plugins") и добавьте к существующей установке пакет "C/C++".

Ниже приводится ряд важных примечаний относительно установки и настройки некоторых компонентов среды IDE NetBeans:

  1. Пакет Apache Tomcat 6.0.18 включен в варианты загрузки "Java" и "All", но по умолчанию не устанавливается ни в одном из этих вариантов. Для установки Apache Tomcat из пакета "Java" или "All" запустите программу установки и выберите "Apache Tomcat 6.0.18" в диалоговом окне "Customize Installation".
  2. Для разработки на C/C++ (применимо для вариантов загрузки "C/C++" и "All") необходимо установить компиляторы и средства. Для получения дополнительной информации см. раздел Установка и настройка поддержки C/C++.
  3. Если требуется использовать диспетчер доступа системы Sun Java, то помимо варианта загрузки "All" для среды IDE необходимо установить комплект SDK для платформы приложений Java, обновление 5 или выше по адресу http://java.sun.com/javaee/downloads/index.jsp. Проверьте, что в состав версии для загрузки входит диспетчер доступа.
  4. Для разработки приложений Java ME для устройств CDC необходимо загрузить и установить инструментарий Sun Java 1.0 для CDC. После установки необходимо зарегистрировать платформу CDC в среде IDE.
  5. Для платформы Mac OS X можно настроить установку среды IDE NetBeans из вариантов загрузки "Java", "Ruby" и "All". Установку среды IDE из загрузок JavaSE, PHP и C/C++ настроить нельзя.

в начало

Начало загрузки

  1. Перейдите к странице загрузки среды IDE NetBeans 6.5.
  2. Выберите соответствующую платформу из раскрывющегося списка в верхней правой части страницы.
  3. Нажмите кнопку "Download", соответствующую выбранному варианту загрузки для установки.
  4. Сохраните файл программы установки.

в начало

Установка программ

Для установки среды IDE следуйте указаниям, приведенным в этом разделе. Данные указания по установке применимы ко всем поддерживаемым платформам. Список поддерживаемых платформ приводится в разделе Заметки к выпуску среды IDE NetBeans 6.5.

ОС Microsoft Windows, Solaris и Linux

Для установки пакета:

  1. После завершения загрузки запустите программу установки.
    • Для Windows исполняемый файл программы установки имеет расширение .exe. Чтобы запустить файл программы установки, дважды щелкните его.
    • Для платформ Solaris и Linux файл программы установки имеет расширение .sh. Для этих платформ необходимо выполнить следующую команду для исполняемых файлов программы установки:
      chmod +x . /<имя-файла-программы-установки>
  2. Для выбора средств и среды исполнения выполните следующие действия на странице "Welcome" мастера установки:
    1. Нажмите кнопку "Customize".
    2. В диалоговом окне "Customize Installation" выберите требуемые варианты.
    3. Нажмите кнопку "OK".
  3. На странице "Welcome" мастера установки нажмите кнопку "Next".
  4. На странице "License Agreement" прочтите лицензионное соглашение, установите флажок для принятия его условий и нажмите кнопку "Next".
  5. На странице установки среды IDE NetBeans выполните следующие действия:
    1. Выберите каталог для установки среды IDE NetBeans по умолчанию или укажите другой каталог.
      Примечание: Каталог установки должен быть пустым, а профиль пользователя, в котором выполняется программа установки, должен предусматривать полномочия на чтение и запись для этого каталога.
    2. Выберите установку комплекта JDK для использования со средой IDE NetBeans по умолчанию или укажите другую установку из раскрывающегося списка.
      Если мастер установки не нашел совместимую установку комплекта JDK для использования со средой IDE NetBeans, значит, комплект JDK не установлен в местоположении по умолчанию. В этом случае укажите путь к установленному комплекту JDK или отмените текущую установку, установите требуемую версию комплекта JDK и перезапустите эту установку.
    3. Нажмите кнопку "Next".
  6. При появлении страницы установки сервера приложений GlassFish V2 UR2 выполните следующие действия:
    1. Выберите каталог для установки сервера по умолчанию или укажите другой каталог.
      Примечание: Выбранный каталог установки должен быть пустым, а профиль пользователя, в котором выполняется программа установки, должен предусматривать полномочия на чтение и запись для этого каталога.
    2. Из списка совместимых установок комплектов JDK выберите комплект JDK для использования сервером приложений либо нажмите кнопку "Browse" для указания другой установки комплекта JDK.
    3. Измените имя пользователя и пароль для домена сервера по умолчанию или выберите значения по умолчанию, а затем нажмите кнопку "Next".
      По умолчанию используется имя пользователя admin и пароль adminadmin.
    4. Проверьте значения портов по умолчанию (HTTP, HTTPS и Admin) для сервера и при необходимости измените их.
    5. Нажмите кнопку "Next".
  7. При появлении страницы установки сервера приложений GlassFish v3 Prelude согласитесь с каталогом установки по умолчанию или укажите другое место установки.
  8. При появлении страницы установки пакета Apache Tomcat выберите каталог установки по умолчанию или укажите другое местоположение для установки.
  9. На странице "Summary" проверьте правильность списка компонентов, которые требуется установить, а также наличие достаточного пространства в системе для установки.
  10. Для запуска процесса установки нажмите кнопку "Install".
  11. После завершения установки:
    1. Если установлен сервер приложений GlassFish, для его регистрации установите флажок "Register the Sun Java System Application Server and NetBeans IDE after finishing the installer".
    2. Если сервер приложений GlassFish не установлен, установите флажок "Register the NetBeans IDE after finishing the installer" для регистрации среды IDE NetBeans.
    3. Для выхода из мастера нажмите кнопку "Finish".
  12. Если на предыдущем этапе был установлен флажок "Register…", в обозревателе откроется страница "Registration". Нажмите кнопку "Register".
  13. Введите учетную запись Sun Online или создайте ее на странице www.sun.com, на которую можно перейти по этой ссылке.

Примечание: При возникновении проблем во время установки пакета см. раздел Устранение неполадок, в котором содержится описание соответствующих процедур и возможных обходов проблем, влияющих на процесс установки.

Примечание: Если сервер приложений GlassFish зарегистрирован после установки некорректно, это можно исправить при запуске среды IDE NetBeans или посредством консоли администратора сервера приложений GlassFish.

Примечание: Если среда IDE NetBeans 6.5 зарегистрирована после установки некорректно, это можно исправить при запуске среды IDE NetBeans.

Mac OS X

Для установки программ:

  1. После завершения загрузки запустите программу установки. Файл программы установки имеет расширение .dmg.
  2. На открывшейся панели щелкните значок пакета. Пакет имеет расширение .mpkg. Запускается мастер установки.
  3. На странице "Welcome" мастера установки нажмите кнопку "Continue".
  4. Прочтите лицензионное соглашение и нажмите кнопку "Continue". Для принятия условий лицензионного соглашения нажмите кнопку "Accept" во всплывающем окне.
  5. На странице "Select a Destination" выберите диск и нажмите кнопку "Continue".
  6. Если были загружены пакеты "All", "Java" или Ruby, можно настроить индивидуальную установку. В нижней левой части последней панели мастера установки нажмите кнопку "Customize". Появится дерево программ.
  7. Выберите программы, которые требуется установить.
  8. Введите соответствующее имя администратора и пароль и нажмите "OK" для запуска установки.

Удаление программ

Microsoft Windows

Если необходимо, можно удалить среду IDE NetBeans, сервер приложений GlassFish V2 UR2, сервер приложений GlassFish v3 Prelude и пакет Apache Tomcat с помощью следующих процедур.

Для удаления среды IDE:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите среду IDE NetBeans 6.5 и нажмите кнопку "Удалить".

Для удаления сервера приложений GlassFish V2 UR2:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите GlassFish V2 UR2 и нажмите кнопку "Удалить".

Для удаления сервера приложений GlassFish v3 Prelude:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите GlassFish V2 UR2 и нажмите кнопку "Удалить".

Для удаления пакета Apache Tomcat:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите Apache Tomcat 6.0.16 и нажмите кнопку "Удалить".

ОС Solaris и Linux

Если необходимо, можно удалить среду IDE NetBeans, сервер приложений GlassFish V2 UR2, сервер приложений GlassFish v3 Prelude и пакет Apache Tomcat с помощью следующих процедур.

Для удаления среды IDE:

  1. Завершите работу среды IDE.
  2. В каталоге установки IDE запустите файл uninstall.sh.
  3. Для удаления GlassFish/Tomcat вместе со средой IDE на странице "Summary" проверьте правильность версий и соответствующих флажков.
  4. Нажмите кнопку "Uninstall" на странице "Summary".
  5. После завершения удаления нажмите кнопку "Finish".

Для удаления сервера приложения GlassFish V2 UR2 или V3 Prelude:

  1. Завершите работу среды IDE.
  2. Определите каталог установки сервера приложений.

    Примечание: Местоположение установки по умолчанию зависит от типа установки и платформы. В следующей таблице приводятся каталоги установки сервера приложений по умолчанию.

    Платформа Установка от имени...

    GlassFish V2 UR2

    Установка сервера приложений

    		 Установка сервера приложений GlassFish v3 Prelude
    ОС Solaris
    		 root /opt/glassfish-v2ur2 /opt/glassfish-v3-prelude
    ОС Solaris,
    Linux    		 user ~/glassfish-v2ur2 ~/glassfish-v3-prelude
    Linux root /usr/local/glassfish-v2ur2 /usr/local/glassfish-v3-prelude
  3. В каталоге установки сервера приложений запустите файл uninstall.sh.
  4. Нажмите кнопку "Uninstall" на странице "Summary".
  5. После завершения удаления нажмите кнопку "Finish".

Примечание: Сервер приложений GlassFish необходим при использовании функциональности SOA. После удаления сервера приложений функциональность SOA автоматически удаляется из установленной среды IDE NetBeans.

Для удаления пакета Apache Tomcat:

  1. Завершите работу среды IDE.
  2. Определите каталог установки пакета Apache Tomcat и запустите файл uninstall.sh.
  3. Нажмите кнопку "Uninstall" на странице "Summary".
  4. После завершения удаления нажмите кнопку "Finish".

Mac OS X

Для удаления среды IDE и сервера приложений:

  1. Завершите работу программного обеспечения.
  2. Перетащите папки или файлы программы в корзину.
    Учтите, что местоположением для среды IDE и сервера приложений по умолчанию является /Applications/NetBeans/.

Обновление и сосуществование предыдущих версий

Одновременно со средой IDE NetBeans 6.5 в одной системе могут существовать выпуски среды IDE NetBeans 5.5, 5.5.1, 6.0 или 6.1. Для установки или запуска среды IDE NetBeans 6.5 удаление версий 5.5, 5.5.1, 6.0 или 6.1 не требуется.

Если у вас установлена среда IDE NetBeans версий 5.5, 5.5.1, 6.0 или 6.1, то вы сможете выбрать, следует ли импортировать пользовательские настройки из существующего пользовательского каталога при первом запуске NetBeans IDE 6.5.

Устранение неполадок

О рабочем каталоге программы установки

Программа установки сохраняет данные о сеансах установки в рабочем каталоге. Эти данные включают список установленных в системе компонентов, сообщения файла регистрации установки и информацию из реестра.

Местоположением рабочего каталога программы установки по умолчанию является %USERPROFILE%\.nbi для ОС Windows и ~/.nbi для других платформ. Файлы регистрации установки содержатся в подкаталоге файла регистрации.

Очистка после незавершенных установок

После незавершенных или неуспешных установок может потребоваться очистка системы.

Для ОС Microsoft Windows, Solaris и Linux:

  1. Удалите рабочий каталог программы установки.
  2. Удалите каталоги установки среды IDE и сервера приложений GlassFish.
  3. На платформе Windows удалите все записи nbi* из следующих разделов системного реестра:
    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall
    HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Для Mac OS X:

  • Перетащите папки или файлы программы в корзину.

Отключение программного обеспечения перед установкой

Перед началом установки рекомендуется отключить брандмауэр, антивирусное программное обеспечение и приложения мониторинга. Некоторые из этих приложений могут стать причиной некорректной настройки компонентов среды IDE в течение процесса установки.

Например, если в системе выполняется приложение монитора процессов Logitech (LVPrcSrv.exe), перед установкой его рекомендуется отключить.
