NetBeans 5.5, Notas de versão do módulo de modelagem UML

Última atualização: 23 de março de 2007

O NetBeansâ„¢ 5.5, Módulo de modelagem UML fornece os recursos de modelagem UML para o NetBeans IDE. A modelagem UML possibilita que analistas e designers criem aplicativos usando uma linguagem de modelagem padrão. Os desenvolvedores estão aptos a gerar o código-fonte Java correspondente para o novo modelo UML e continuar com o desenvolvimento do aplicativo. Usando os recursos Gerar código e Reverter engenharia, as modificações no modelo UML ou no código-fonte Java podem ser refletidas no código-fonte ou no modelo, respectivamente.

Esta documentação de notas de versão fornece algumas informações importantes que você deve saber sobre a versão do NetBeans 5.5, Módulo de modelagem UML.

Requisitos do sistema

Seu sistema deve atender aos requisitos especificados na seção Requisitos do sistema das Notas de versão do NetBeans IDE 5.5, com a exceção dos requisitos da plataforma Macintosh OS X. O módulo de modelagem UML atualmente não é compatível com a plataforma Macintosh OS X. Consulte as Notas de versão da plataforma Mac OS para obter mais informações sobre os problemas e soluções conhecidos do NetBeans 5.5, Módulo de modelagem UML.

Problemas conhecidos

A versão do NetBeans 5.5, Módulo de modelagem UML, permite que você explore e avalie a funcionalidade UML no IDE. Pedimos que você participe e nos dê a sua opinião, registrando qualquer problema que tenha encontrado no sistema de rastreamento de problemas em https://netbeans.org/community/issues.html.

Os problemas não solucionados desta versão são os seguintes:

Os problemas descritos acima ocorrem quando o UML é usado no IDE que está sendo executado em todas as plataformas suportadas.

  • Quando se faz a engenharia reversa em um projeto Java, o tipo Genérico de atributos, operações e operações em classes aninhadas é perdido.

    Descrição:

    • Erro #87639: O tipo Genérico de atributos, operações e operações em classes aninhadas é perdido quando é feita a engenharia reversa de um código-fonte Java em um projeto UML.

    Solução: Nenhum.

  • Classes aninhadas que foram removidas do código-fonte ainda aparecem no modelo UML após uma ação de engenharia reversa.

    Descrição:

    Erro #87841: Se uma classe aninhada for removida de um projeto Java e o projeto Java tiver sofrido engenharia reversa no mesmo projeto UML anterior, a classe aninhada que foi removida ainda aparecerá no modelo UML no Editor de diagramas.

    Solução: Feche e reabra o diagrama na janela do Editor de diagramas.

  • A entrada de caracteres multibyte em qualquer nome de elemento em qualquer diagrama fica corrompida quando é usado o editor em linha desse elemento.

    Descrição:

    Erro #78353: Ao usar o editor em linha para inserir caracteres multibyte de qualquer nome de elemento em qualquer diagrama, os caracteres multibyte resultantes ficam corrompidos.

    Solução:

    Em vez de usar o editor em linha, use o editor de propriedades para inserir os caracteres multibyte de qualquer nome de elemento em qualquer diagrama.
  • Se o IDE tiver sido iniciado com as configurações de memória padrão, um erro de memória talvez ocorra durante o uso dos recursos de modelagem UML com projetos Java ou UML grandes.

    Descrição:

    Erro #86049: Depois de uma prolongada atividade do IDE, a configuração de memória padrão do IDE -J-Xmx128m talvez se torne insuficiente durante a realização de tarefas de modelagem UML em projetos grandes de Java ou de modelagem UML. Uma exceção OutOfMemory pode ocorrer.

    Solução:

    Substitua a configuração de memória padrão -J-Xmx128m por um valor -J-Xmx512m na declaração netbeans_default_options do arquivo diretório_de_instalação_do_nb5.5/etc/netbeans.conf e reinicie o IDE.

O problema descrito acima ocorre quando o UML é usado no IDE que está sendo executado nas plataformas Solaris e Linux.

  • O desempenho da modelagem UML é lento em uma área de trabalho remota.

    Descrição:

    • Erro #78312: Pode-se demorar 30 vez mais para realizar relatórios da Web em uma área de trabalho remota
    • Erro #78574: Até mesmo quando um pequeno diagrama é exportado para um formato .png, o IDE congela quando está sendo executado em uma área de trabalho remota.
    • Erro #78505: O desempenho da janela Visão geral é lento em uma área de trabalho remota.

    Solução:

    Para aprimorar o desempenho do recurso de modelagem UML quando ele é usado em sistemas remotos, adicione a opção da linha de comando -J-Dsun.java2d.pmoffscreen=false na netbeans_default_options declaração no arquivo
    nb5.5-installation-directory/etc/netbeans.conf e reinicie o IDE.

O problema descrito acima ocorre quando o UML é usado no IDE que está sendo executado nas plataformas Windows, Solaris e Linux.

  • Os nomes de elemento são perdidos quando os diagramas de modelagem são impressos no modo paisagem e alguns caracteres de lixo são mostrados.

    Descrição:

    Erro #81127: Os diagramas impressos no modo paisagem não possuem nomes de elemento em qualquer elemento e alguns caracteres de lixo estão aparecendo. Os nomes de componente do elemento são deslocados para baixo de cada elemento.

    Solução: Esses problemas ocorrem somente durante o uso do IDE com Java SE 1.6. Reinicie o IDE usando o JDK 1.5.

Documentação

A documentação desta versão do NetBeans 5.5, Módulo de modelagem UML, encontra-se na forma de tutoriais com base na Web e do sistema de ajuda do IDE. Você pode acessar as páginas da ajuda on-line da modelagem UML, escolhendo Ajuda > Conteúdo da Ajuda na barra de ferramentas principal do IDE e selecionando as páginas sob o tópico Modelagem UML. Os tutoriais podem ser encontrados na seção de tutoriais do site da documentação do módulo de modelagem UML.
