Notas da versão do NetBeans™ C/C++ Development Pack 5.5.1 atualização 1

Bem-vindo ao NetBeans C/C++ Development Pack.

O NetBeans™ C/C++ Development Pack 5.5.1 atualização 1 permite desenvolver aplicativos em C/C++ com NetBeans IDE em sistemas operacionais Microsoft Windows, Linux e Solaris.

O NetBeans C/C++ Development Pack 5.5.1 atualização 1 é compatível com NetBeans IDE 5.5.1 e NetBeans IDE 5.5.

Requisitos do sistema

Sistemas operacionais suportados

O NetBeans C/C++ Development Pack 5.5.1 atualização 1 é executado em sistemas operacionais que oferecem suporte à máquina virtual Java. O NetBeans C/C++ Development Pack 5.5.1 atualização 1 foi testado nos seguintes sistemas operacionais:

  • Microsoft Windows XP Professional SP2
  • Red Hat Fedora Core 3
  • Solaris™ 10 Operating System (Solaris OS) (SPARC® e x86/x64 Platform Edition)
  • Mac OS X versão 10.4 e superior

O NetBeans C/C++ Development Pack também pode ser executado nas seguintes plataformas:

  • Microsoft Windows 2000 Professional SP4
  • Solaris™ 8 OS (SPARC® e x86/x64 Platform Edition) e Solaris™ 9 OS (SPARC® e x86/x64 Platform Edition)
  • Várias distribuições do Linux

Informações de configuração

O NetBeans IDE apresenta um tamanho de pilha padrão de 128 MB. O NetBeans C/C++ Development Pack é executado corretamente com esta configuração padrão no desenvolvimento de projetos pequenos com até 500 arquivos de código e cabeçalho.

Ao desenvolver projetos grandes, será necessário aumentar o tamanho de pilha. Se for exibida uma exceção OutOfMemory durante o desenvolvimento de um projeto grande, a causa de tal exceção será provavelmente o tamanho de pilha.

Você pode definir o tamanho de pilha na Java Virtual Machine (JVM)* na qual o NetBeans IDE é executado no arquivo netbeans.conf.

Para alterar o tamanho de pilha:

  1. Copie o netbeans.conf da pasta etc do diretório de instalação do NetBeans para o diretório de usuário do NetBeans ($HOME/.netbeans/5.5.1).
  2. No arquivo netbeans.conf do diretório de usuário, edite as opções de inicialização de Java da linha de comando -J-Xmx (em negrito abaixo) no arquivo netbeans.conf e reinicie o IDE. 
    # command line switches
netbeans_default_options="-J-Xms32m -J-Xmx128m -J-XX:PermSize=32m
-J-XX:MaxPermSize=96m -J-Xverify:none -J-Dapple.laf.useScreenMenuBar=true"

Os tamanhos de pilha recomendados para aplicativos médios e grandes do NetBeans C/C++ Development Pack são:

  • Para desenvolver aplicativos médios (de 500 a 2000 arquivos de código e cabeçalho) em um sistema com 1 GB ou mais de RAM: 512 MB
  • Para desenvolver aplicativos grandes (mais de 2000 arquivos de código e cabeçalho) em um sistema com 2 GB ou mais de RAM: 1 GB

Não aumente o tamanho de pilha em mais de 1 GB.

Se estiver executando a Sun JVM, você também pode adicionar as opções de coletor de lixo -J-XX:+UseConcMarkSweepGC (coletor atual) e -J-XX:+UseParNewGC (coletor paralelo) ao arquivo netbeans.conf. Estas opções permitem que o coletor de lixo seja executado junto com o mecanismo de execução principal. Outras implementações diferentes da Sun JVM podem não oferecer suporte a tais opções.

Para obter mais informações sobre as configurações de opções do NetBeans, consulte Configurando a seção IDE de Usando o NetBeans IDE 5.5

Para obter mais informações sobre o ajuste de desempenho do NetBeans, consulte Ajustando as opções da para melhorar o desempenho.

* Observação: Os termos "Java Virtual Machine" e "JVM" referem-se a uma máquina virtual da plataforma Java(TM).

Problemas conhecidos

Esta versão do NetBeans C/C++ Development Pack 5.5.1 atualização 1 permite explorar e avaliar a nova funcionalidade no IDE. Pedimos que você participe e nos dê a sua opinião, registrando qualquer problema que tenha encontrado no sistema de rastreamento de problemas em https://netbeans.org/community/issues.html.

Os problemas não solucionados desta versão são os seguintes:

  • Erro #89880: O projeto não é reprocessado quando o conjunto de compiladores é alternado

    Descrição: Se alternar os conjuntos de compiladores de um projeto, o projeto não é reprocessado. Após a alteração, o auto-completar de código e os hyperlinks podem não funcionar corretamente no projeto.

    Solução: Feche o projeto e abra-o novamente.

  • Erro #76173: Não há esvaziamento automático da saída antes da entrada

    Descrição: Se a propriedade padrão Tipo de console do projeto for alterada de Terminal externo para Janela de saída (somente saída), nenhuma entrada poderá ser digitada na janela Saída.

    Solução: Na categoria Execução das propriedades do projeto, altere a propriedade padrão Tipo de console apenas dos aplicativos somente de saída.

  • Erro #89343: A pausa não funciona no Windows quando o programa espera a entrada do usuário

    Descrição: Clicar em Pausa enquanto o programa está esperando a entrada não para o programa. A pausa funciona somente depois ter fornecido a entrada.

    Solução: Não há solução.

  • Erro #90120: "pausa" (no Windows) não funciona em casos extremos

    Descrição: Além do 89343, há vários outros casos nos quais Pausa não funciona no Windows. Um destes casos é em loops muito fechados.

    Solução: Não há solução.

  • Erro #90222: Os pontos de interrupção não funcionam imediatamente durante a execução

    Descrição: Se um ponto de interrupção for definido durante o processo de execução, tal ponto não é passado para o gdb até a próxima interrupção do processo.

    Solução: Pause o processo, defina o ponto de interrupção e prossiga.
