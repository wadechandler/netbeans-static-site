NetBeans Mobility Pack for CDC 5.5.1 发行说明

欢迎使用 NetBeans Mobility Pack for CDC，此附加软件包提供了一些工具，可用于创建支持 Java Micro Edition (Java ME) 联网的设备配置 (Connected Device Configuration, CDC) 技术的 Java ME 应用程序。

系统要求

支持的操作系统

NetBeans Mobility Pack for CDC 5.5.1 在 Java SE JDK 5.0 和 6.0 版本上运行，该软件由 Java Runtime Environment 和开发工具（用于编译、调试和运行使用 Java 语言编写的应用程序）构成。

NetBeans Mobility Pack for CDC 在支持 Java VM 的 Microsoft Windows 系统上运行。下面列出了已通过 NetBeans IDE 测试的平台。

  • Microsoft Windows XP Professional SP2
  • Windows Vista
  • Ubuntu 6.x
最低的硬件配置

Mobility Pack for CDC 需要的资源并不明显多于 NetBeans IDE 所需的资源。

注意：IDE 要求的最低屏幕分辨率为 1024x768 像素。

  • 处理器：500 MHz Intel Pentium III 工作站或与之等效的处理器
  • 内存：512 MB
  • 硬盘空间：850 MB 可用硬盘空间
建议的硬件配置
  • 处理器：1.4 GHz Intel Pentium III 工作站或与之等效的处理器
  • 内存：1 GB
  • 硬盘空间：1 GB 可用硬盘空间
所需的软件

NetBeans IDE 在 J2SE JDK (Java SE Development Kit) 5.0 和 6.0 版本上运行，该软件由 Java Runtime Environment 和开发工具（用于编译、调试和运行使用 Java 语言编写的应用程序）构成。

您可以访问 http://java.sun.com/javase/downloads/index.jsp 站点，下载适用于您的平台的 JDK。

仿真器平台

NetBeans Mobility Pack 中集成了 Sun Java Toolkit for CDC。但是，如果要为其他支持的平台编写应用程序，则需要下载并安装以下一个或多个 SDK：

  • Ricoh
    • Embedded Software Architecture Emulator（下载时要求注册）
  • Sony Ericsson
    • UIQ 3 SDK（下载时要求登录）
    • M600 Extension Package for the UIQ 3 SDK Beta 3（下载）
    • P990 Extension Package for the UIQ 3 SDK Beta 3（下载）
  • Nokia
    • Nokia Series 80 Platform SDK for Symbian OS, for Java, Personal Profile（下载）
  • Windows Mobile with NSIcom CrEme VM
    • CrEme 4.10 Developer Support（下载）
    • Microsoft Device Emulator 1.0（下载）
    • Virtual Machine Network Driver for Microsoft Device Emulator（下载)
    • ActiveSync（下载）
  • SavaJe
    • SavaJe Mobile Platform（不再提供）

 

CDC Pack 5.5.1 支持部署到 Windows Mobile（带有 NSIcom VM）和 SavaJe 设备上。

NetBeans Mobility Pack for CDC 5.5.1 安装指南提供了安装各种仿真器的说明。

兼容性

首次运行带有 NetBeans Mobility Pack 的 IDE 时，可以导入在 NetBeans Mobility Pack for CDC 5.5 中使用的一部分设置。如果您选择不导入早期发行版本中的设置，则使用缺省设置打开 IDE。

注意：IDE 只能识别那些用户目录位于缺省位置的早期安装。它不能识别您使用 --userdir 开关在其中指定了用户目录的安装。如果您要从安装程序无法识别的 IDE 中导入设置，请下载 IDE 已归档的版本（而不是安装程序）。

尽管可以将 NetBeans Mobility Pack for CDC 5.0 的设置导入到 NetBeans 5.5.1 中，但是无法将 NetBeans Mobility Pack for CDC 5.5.1 的设置导入到早期的 IDE 发行版本中。

已知问题

以下是本发行版本中存在的一些待解决的问题：

  • 问题 #77083：icon.small/icon.focused 在部署后具有绝对路径

    描述：bundle.jnlp 包含 icon.small 和 icon.focused 的绝对路径。这是因为图标位于设备上。在文本字段中，您必须按照在设备上所定义的位置，输入图标的路径。设备文档中会列出正确位置（路径）。

  • 问题 #86566：保存组件面板设置时出现错误。

    描述：如果更改项目的仿真器平台，则不会保存 GUI 设计器组件面板设置。

  • 问题 #88709：在仿真器平台之间切换时，生成失败。

    描述：如果将平台由 Nokia 更改为 Sony Ericsson，则生成将会失败。

    解决方法：通过右键单击项目并选择“项目属性”>“常规”> "Sony Ericsson" 来更改 UID。

  • 问题 #77083：bundle.jnlp 文件中包含图标的绝对路径链接。

    描述：icon.small 和 icon.focused 在部署后具有绝对路径。

    解决方法：根据设备文档来指定设备的路径。

  • 问题 #88913：找不到 nbproject/build-impl.xml 错误。

    描述：如果生成、运行和重命名项目，IDE 的国际化版本将收到错误消息。

  • 问题 #90047：在调试 CDC 项目时出现问题。

    描述：在尝试调试 CDC 项目时抛出 java.lang.NoClassDefFoundError。

    解决方法：如果将 CDC Toolkit 项目装入到 Mobility Pack for CDC 中，则调试将无法正常工作。您必须将项目升级到 Mobility Pack for CDC，方法是：转至项目的属性并单击“确定”，然后按照 IDE 提供的步骤执行操作。

  • 问题 90647：无法在 IDE 中更改 Sony Ericsson UIQ 平台的设备。

    描述：无法在 IDE 中更改平台的目标设备。

    解决方法：运行位于 <install_dir>\UIQ3SDK\epoc32\tools\distrib\ 中的 SDKConfig.exe 实用程序，其中 <install_dir> 是计算机上 Sony Ericsson UIQ3 仿真器的安装目录。

  • 问题 #90654：在升级到 FCS 版本后，使用 Mobility Pack for CDC 5.5.1 update1 生成项目时失败。

    描述：问题是由于 CDC 软件包 FCS 版本的执行方法不正确而造成的。

    解决方法：在升级通过 Mobility Pack for CDC 5.5.1 update1 创建的项目（使用 AGUI）时，将项目属性更改为 Xlet 执行方法。
