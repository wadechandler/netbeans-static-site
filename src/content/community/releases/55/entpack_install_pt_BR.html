Instruções de instalação do NetBeans Enterprise Pack 5.5

O NetBeans Enterprise Pack 5.5 inclui módulos de ferramentas do esquema XML, módulos BPEL, módulos de serviços de segurança da Web, Sun Java System Application Server Platform Edition 9 atualização 1, Project Open ESB Starter Kit, Sun Java System Access Manager 7.1 e Sun Java System Policy Agent 2.2 for Sun Web Services.

Observação: A funcionalidade UML já não está disponível na versão final (primeira remessa a clientes) para download do NetBeans Enterprise Pack 5.5. Mas é oferecida separadamente na Central de atualizações automáticas.

Pré-requisitos

Esse tópico especifica os pré-requisitos que devem ser cumpridos antes de iniciar a instalação do NetBeans Enterprise Pack 5.5.

Software necessário

Antes da instalação do NetBeans Enterprise Pack 5.5, o software abaixo deve ser instalado:

  • NetBeans IDE 5.5 (daqui em diante denominado IDE)

  • O JDK 5.0 atualização 9 (versão 1.5.0_09; daqui em diante denominado JDK) ou superior deve ser instalado. Se tiver o JDK versão 5.0 atualização 9 ou superior instalado, não é possível prosseguir com a instalação. Para obter detalhes, consulte o tópico Software necessário em Notas da versão.

    Observação: Para Mac OS X, é necessário ter como mínimo o JDK 5.0 versão 3.

Aplicativos firewall

Os aplicativos firewall ativados podem impedir que alguns componentes sejam configurados corretamente durante o processo de instalação. Desative os aplicativos firewall antes iniciar a instalação.

Informações de configuração

Este tópico oferece informações sobre a configuração que podem ser necessárias para configurar o software.

Servidor de aplicativo Sun Java System

Os valores da senha e do nome de usuário padrão do domínio padrão do servidor de aplicativos, domain1, são os seguintes:
nome de usuário: admin
senha: adminadmin

Gerenciador de acesso Sun Java System

Se precisar acessar o console da administração do servidor do gerenciador de acesso, use os valores seguintes:
nome de usuário: amadmin
senha: admin123

Instalação do software

Essa seção apresenta as instruções para a instalação do NetBeans Enterprise Pack 5.5 em todas as plataformas suportadas.

Windows

Encontram-se abaixo as instruções para a instalação do NetBeans Enterprise Pack 5.5 na plataforma Microsoft Windows.

Antes de começar

A instalação na plataforma Microsoft Windows apresenta as seguintes restrições:

  • O instalador extrai os arquivos necessários para o diretório temporário padrão definido pela variável de ambiente TEMP. Se o espaço em disco no diretório temporário é menor que o necessário, a instalação pode falhar ou ser feita de forma incorreta.
    Se tiver este problema, você pode aumentar o espaço alocado da variável de ambiente TEMP ou especificar outro diretório temporário com espaço suficiente. Para especificar um diretório temporário para a instalação, use os seguintes comandos de uma janela de comando:

    > cd /d  local-do-instalador
    > netbeans-ent_pack-5_5-windows.exe -is:tempdir diretório-temporário

Para instalar

  1. Depois de fazer o download do arquivo de instalação, netbeans-ent_pack-5_5-windows.exe, clique duas vezes no ícone do instalador para iniciá-lo.

  2. Na página de boas-vindas do assistente de instalação, clique em Próximo.

  3. Responda o contrato de licença e clique em Próximo.
    A página de seleção da pasta de instalação do NetBeans IDE 5.5 do assistente se abre.

  4. Na lista de instalações compatíveis, escolha o IDE no qual o NetBeans Enterprise Pack será instalado e clique em Próximo.
    Também é possível digitar o caminho de outro IDE instalado ou usar o botão Explorar para ir até outra instalação e selecioná-la.

  5. Selecione os componentes do NetBeans Enterprise Pack que deseja instalar e clique em Próximo.
    Observação: Desmarque as caixas de seleção BPEL e Serviços de segurança da Web se não tiver a intenção de usar esses módulos. Isto ativa o botão de opção Ignorar servidor de aplicativos na etapa seguinte e acelera o processo de instalação.
    Observação: Certifique-se de observar as dependências listadas na tabela abaixo ao selecionar os componentes para instalação.

    Nome do componente necessário Nome do componente dependente
    Componente XML do NetBeans Enterprise Pack 5.5 Componente BPEL do NetBeans Enterprise Pack 5.5
    Sun Java System Application Server Platform Edition 9

    Os componentes do NetBeans Enterprise Pack 5.5 seguintes:

    • BPEL
    • Serviços de segurança da Web


  6. Adote um dos seguintes procedimentos:

    • Para instalar o servidor de aplicativos incorporado, siga as instruções em Para continuar a instalação com o servidor de aplicativos incorporado.

    • Se tiver um servidor de aplicativos Sun Java System Application Server Platform Edition 9 ou do Project Glassfish pré-instalados siga as instruções em Para continuar a instalação com o servidor de aplicativos pré-instalado.

Para continuar a instalação com o servidor de aplicativos incorporado

  1. Na página das opções de instalação do NetBeans Enterprise Pack do assistente de instalação, selecione o botão de opção Instalar o servidor de aplicativos incorporado e clique em Próximo.

  2. Na página da pasta de instalação do servidor de aplicativos do assistente de instalação, aceite a pasta de instalação padrão do servidor de aplicativos, especifique outra pasta, esvazie a pasta de instalação ou use o botão Explorar para ir até a pasta de instalação e selecioná-la, e clique em Próximo.

  3. Na lista de instalações compatíveis, escolha o JDK que o servidor de aplicativos irá usar e clique em Próximo.
    Você também pode digitar o caminho de outro JDK instalado ou usar o botão Explorar para ir até outra instalação e selecioná-la.

  4. Defina o nome de usuário e a senha do domínio padrão do servidor de aplicativos, domain1, ou aceite os padrões e clique em Próximo.
    O nome de usuário e a senha padrão são admin e adminadmin.

  5. Verifique os valores da porta (Admin, HTTP e HTTPS) do domínio padrão do servidor de aplicativos e altere-os se necessário. Em seguida clique em Próximo.

  6. Verifique se a lista dos componentes que serão instalados está correta e se há espaço suficiente no sistema para a instalação.

  7. Clique em Instalar para iniciar a instalação.

  8. Quando a instalação estiver concluída, clique em Finalizar para sair do assistente.

Para continuar a instalação com o servidor de aplicativos pré-instalado

  1. Na página das opções de instalação do NetBeans Enterprise Pack do assistente de instalação, selecione botão de opção Instalar o servidor de aplicativos pré-instalado e clique em Próximo.

  2. Na página da pasta de instalação do servidor de aplicativos do assistente de instalação, escolha o servidor de aplicativos que deseja usar na lista de instalações compatíveis ou use o botão Explorar para ir até outra pasta de instalação de servidor de aplicativos e selecioná-la, e clique em Próximo.

  3. Se o domínio padrão, domain1, já existir no servidor de aplicativos selecionado, insira o nome de usuário e a senha de tal domínio e clique em Próximo.
    O assistente de instalação verifica as informações introduzidas do nome de usuário e da senha.
    Observação: O processo de verificação pode demorar alguns minutos.

    -ou-

    Se o domínio padrão, domain1, ainda não existir no servidor de aplicativos selecionado, adote o seguinte procedimento:

    1. Defina o nome de usuário e a senha do domínio padrão do servidor de aplicativos, domain1, ou aceite os padrões e clique em Próximo.
      O nome de usuário e a senha padrão são admin e adminadmin.

    2. Verifique os valores da porta (Admin, HTTP e HTTPS) do servidor de aplicativos e altere-os se necessário. Em seguida clique em Próximo.


  4. Verifique se a lista dos componentes que serão instalados está correta e se há espaço suficiente no sistema para a instalação.

  5. Clique em Instalar para iniciar a instalação.

  6. Quando a instalação estiver concluída, clique em Finalizar para sair do assistente.

Observação: Se tiver problemas para concluir com sucesso a instalação do software, consulte Soluções de problemas para obter descrições completas e soluções sugeridas para problemas não solucionados que poderiam afetar o processo de instalação.

Desinstalação do software

Se tiver instalado os servidor de aplicativos incorporado, a desinstalação remove o NetBeans Enterprise Pack 5.5, o Sun Java System Application Server Platform Edition 9, o Project Open ESB Starter Kit e Sun Java System Access Manager do sistema. Se tiver preferido usar o servidor de aplicativos pré-instalado, a desinstalação remove o NetBeans Enterprise Pack 5.5 do sistema. Neste caso, para remover o Project Open ESB Starter Kit, você deve desinstalar o servidor de aplicativos pré-instalado.

Microsoft Windows

Para desinstalar o software

  1. Feche o IDE.

  2. No Painel de controle, selecione Adicionar ou remover programas, em seguida, selecione NetBeans Enterprise Pack 5.5 e clique em Alterar/Remover.

  3. Siga as instruções no desinstalador.

Solaris OS, Linux

Para desinstalar o software

  1. Feche o IDE.

  2. Navegue até a subpasta bin/_uninst_entpack da pasta de instalação do NetBeans Enterprise Pack 5.5.

  3. Execute o script uninstall.sh.

Mac OS X

Para desinstalar o software

  1. Feche o IDE.

  2. Navegue até a subpasta bin/_uninst_entpack da pasta de instalação do NetBeans 5.5.

  3. Execute o arquivo executável uninstall.jar.
    Observação: Se a versão padrão de Java do seu sistema for 1.4, não será possível iniciar a desinstalação com um duplo clique no arquivo .jar. Nesse caso, execute o arquivo .jar manualmente a partir da linha de comando que usa o Java 1.5.

Solução de problemas

Se tiver problemas ao instalar ou executar o software, consulte a lista de problemas abaixo para ver as soluções. Para obter uma lista completa dos problemas conhecidos, dos requisitos do sistema e outras informações úteis, consulte Notas da versão do NetBeans Enterprise Pack 5.5.

Encontram-se abaixo os problemas não solucionados desta versão que podem afetar ou interromper o processo de instalação:

  • O instalador não se inicia corretamente com pouco ou nenhum espaço livre em disco no diretório temp.

    Descrição: No Microsoft Windows, se o sistema estiver configurado para usar um diretório temporário no drive C e este diretório temporário não tiver espaço suficiente para os arquivos temporários que o instalador necessita, o instalador notifica que não há espaço em disco suficiente no drive C, mesmo se você tiver optado por realizar a instalação em outro drive. O instalador está, na verdade, avisando que necessita mais espaço temporário.
    O problema encontra-se na estrutura do InstallShield.

    Solução: Inicie o instalador com o parâmetro de linha de comando -is:tempdir dir-temp-personalizado. O dir-temp-personalizado deve ter como mínimo 130 MB de espaço livre em disco.

  • NoClassDefFoundError: não foi possível acessar amserver ocorre durante a instalação.

    Descrição: Se tentar instalar o software pela segunda vez sem limpar o diretório do gerenciador de acesso, a configuração do gerenciador de acesso pode falhar durante a instalação.

    Solução:

    • No Microsoft Windows:
      Exclua o diretório especificado: %SystemDrive%\Documentos e configurações\nome-de-usuário\AMConfig_server_amserver_ e remova o arquivo %SystemDrive%\Documentos e configurações\nome-de-usuário\AMConfig_server_amserver_.

    • No UNIX/Mac:
      Exclua o diretório especificado: $HOME/AMConfig_server_amserver_ e remova o arquivo $HOME/AMConfig_server_amserver_.
  • O instalador falha ao usar as opções -console e -silent.

    Descrição: Nas plataformas UNIX, se o instalador não puder executar o processo de instalação sem o modo de gráficos, o próprio instalador sugere que você o execute com as opções -console e -silent. Ao tentar fazer desta maneira, o instalador falha.

    Solução: Ignore a mensagem do instalador e configura o ambiente de GUI conforme descrito abaixo para continuar a instalação.
    Este problema se deve na maioria das vezes a que a conexão ao destino está sendo realizada via telnet/ssh/rlogin, o que quer dizer que você deve redirecionar a GUI ao sistema original. Para ter sucesso, os dois requisitos seguintes devem ser cumpridos:

    1. Um servidor X deve estar configurado e em execução no sistema a partir do qual está realizando a conexão.

    2. A variável de ambiente DISPLAY deve estar definida com o valor correto no sistema de destino. Adotando um shell bash, o valor e o comando corretos da variável são:
      export DISPLAY=nome-da-máquina:número-da-exibição
      Em que número-da-máquina é o endereço IP do sistema no qual está se conectando e número-da-exibição é o número da exibição, geralmente 0.0.
  • A instalação falha após a desinstalação manual.

    Descrição: Se tentar instalar o produto após desinstalar manualmente removendo arquivos, a instalação falhará porque os dados no registro VPD (dados vitais do produto) do InstallShield não foram apagados.

    Solução: Antes de iniciar uma nova instalação, remova o registro VPD do InstallShield. Este registro encontra-se em %SystemDrive%\Arquivo de programas\Arquivos comuns\InstallShield\Universal ou em ~/InstallShield.

  • No Mac OS X, não é possível iniciar a instalação clicando duas vezes no arquivo .jar.

    Descrição: Se a versão padrão de Java do seu sistema for 1.4, não será possível iniciar a instalação com um duplo clique no arquivo .jar.

    Solução: Nesse caso, execute o arquivo .jar manualmente a partir da linha de comando que usa o Java 1.5.

  • Há exceções no instalador ao executar em JRE 1.4.

    Descrição: No Microsoft Windows, se não houver um JDK compatível pré-instalado na máquina de destino, o iniciador InstallShield Windows permite selecionar um java.exe personalizado. Após selecionar um java.exe personalizado o instalador não realiza validação de compatibilidade. Isso significa que se selecionar um java.exe de um JRE 1.4, que não é compatível com o instalador, você encontrará várias exceções. O problema encontra-se na estrutura do InstallShield.

    Solução: Selecione um java.exe de um JRE 1.5. Consulte Software necessário para obter informações sobre o download e a instalação de um JDK compatível.

  • O instalador não se inicia corretamente no UNIX.

    Descrição: O instalador apresenta falhas ao se iniciar corretamente no UNIX quando há pouco ou nenhum espaço em disco no diretório raiz do sistema de arquivos. O problema encontra-se na estrutura do InstallShield. Este problema foi observado em máquinas com menos de 2,5 MB livres na raiz do sistema de arquivos.

    Solução: Libere suficiente espaço em disco na raiz do sistema de arquivos e reinicie o instalador.
