Notas de la versión de NetBeans IDE 6.8

Última actualización: July 10, 2013

NetBeans IDE es un entorno de desarrollo integrado (IDE) modular y basado en estándares, escrito con el lenguaje de programación Java. El proyecto de NetBeans consta de un IDE de código abierto con gran variedad de funciones escrito con el lenguaje de programación Java y una plataforma para aplicaciones de cliente enriquecidas que se puede utilizar como marco genérico para crear cualquier tipo de aplicación. Para obtener información acerca de cómo instalar el software en el sistema, consulte las Instrucciones de instalación de NetBeans IDE 6.8 . Para obtener información sobre las nuevas funciones incluidas en esta versión del IDE, consulte la páginas Información de la versión de NetBeans IDE 6.8 .

Novedades de la versión 6.8

NetBeans IDE 6.8 es una actualización de NetBeans IDE 6.7.1 e incluye los siguientes cambios:

  • Compatibilidad con Java EE 6
  • Compatibilidad con JSF 2.0 y con muchas de las funciones nuevas
  • Compatibilidad con PHP 5.3 y con Symfony Framework
  • Integración mejorada de la función Kenai y compatibilidad con el sistema de seguimiento de incidencias Jira
  • Mejoras adicionales incluidas en NetBeans IDE 6.8

Para obtener más información sobre esta versión, consulte la página Información de la versión de NetBeans IDE 6.8.

Información sobre la compatibilidad con varios idiomas

NetBeans IDE está disponible en las siguientes traducciones compatibles con Sun: portugués de Brasil (BR), japonés (JP) y chino simplificado (ZH).

Las traducciones del IDE realizadas por la comunidad también están disponibles en varios idiomas adicionales y se pueden descargar desde la sección de contribuciones de la comunidad del menú desplegable Idioma del IDE.

Compatibilidad

La primera vez que ejecuta el IDE, puede importar un subconjunto de los valores utilizados en las versiones anteriores de NetBeans IDE. Si decide no importarlo, el IDE se abre con la configuración predeterminada.

Nota: el IDE sólo reconoce instalaciones anteriores en las que el directorio de usuario resida en la ubicación predeterminada. No reconoce las instalaciones en los que se ha especificado un directorio de usuario mediante la instrucción switch --userdir. Si desea importar la configuración de un IDE que el instalador no reconoce, inicie el IDE y use la función Importar de la ventana Opciones. Tras hacer clic en Importar en dicha ventana, utilice el botón Examinar y busque el directorio de usuario (userdir) de la instalación del IDE que contiene la configuración que desea importar. Para obtener más información sobre la ubicación del directorio de usuario de NetBeans IDE, consulte la FAQ ¿Qué es un directorio de usuario y dónde se puede encontrar? .

Nota: si edita las propiedades del proyecto de NetBeans IDE 6.8 en un proyecto creado en NetBeans IDE 4.1 o anterior, el proyecto dejará de funcionar en las versiones de NetBeans IDE anteriores.

Problemas conocidos: instalador

Éstos son los problemas no resueltos del instalador en esta versión:

  • Problema nº 177872: es posible que en Mac OS X, el complemento y las herramientas de actualización de GlassFish v3 no funcionen correctamente

    Descripción: en Mac OS X, el complemento basado en pkgy las herramientas de actualización no funcionan correctamente con cuentas no raíz si GlassFish v3 se instaló en raíz.

    Solución: utilice el comando chown -R para cambiar la propiedad de la instalación de GlassFish y configúrela en el usuario no raíz. Por ejemplo: sudo chown -R <nuevousuario>:admin /Aplicaciones/NetBeans/glassfish-v3-b73, donde nuevousuario es el usuario no raíz. Después de utilizar este comando, el usuario puede ejecutar directamente las herramientas bin/updatetool y bin/pkg herramientas.

  • Problema nº 6720021: el instalador no funciona al utilizar OpenSolaris con cualquier versión de JDK 5.

    Descripción: el instalador no funciona al utilizar OpenSolaris con cualquier versión de JDK 5.

    Solución: instálelo con JDK 6 o ejecute el comando "export AWT_TOOLKIT=XToolkit" antes de ejecutar el instalador con JDK 5.

  • Problema nº 93568: en Solaris y Linux la instalación se bloquea cuando se pierde la conexión de red.

    Descripción: en plataformas Solaris y Linux, si se pierde la conexión de red, la instalación se bloquea cuando el instalador busca un JDK compatible en los directorios de red.

    Solución: restaure la conexión de red y continúe con la instalación.

  • Problema nº 117172: aparece una ventana en blanco al ejecutar el instalador en Ubuntu.

    Descripción: el instalador aparece como un cuadro de diálogo en blanco/gris al ejecutarse en Ubuntu si Beryl/Compiz/Compiz Fusion están instalados.

    Solución: ejecute el comando export AWT_TOOLKIT=MToolkit antes de ejecutar el instalador.

  • Problema nº 123300. En Fedora Core 8, el instalador deja de mostrar la interfaz y vuelve a la consola.

    Descripción: en Fedora Core 8, la interfaz desaparece y el sistema vuelve a la consola.

    Solución: consulte https://netbeans.org/servlets/ReadMsg?list=nbusers&msgNo=98765.

  • El asistente de instalación de NetBeans IDE 6.8 no se inicia correctamente (se bloquea). Ocurre después de que el usuario detenga el proceso de instalación manualmente y el programa de instalación le informe de que hay otra instancia del instalador en ejecución.

    Descripción: en Windows, si se utiliza Java6 de forma predeterminada y se almacenan archivos zip de gran tamaño en el escritorio, el asistente de instalación no se inicia correctamente (se bloquea). Ocurre después de que el usuario detenga el proceso de instalación manualmente y el programa de instalación le informe de que hay otra instancia del instalador en ejecución.

    Solución: antes de la instalación, elimine todos los archivos zip de gran tamaño del escritorio.

Problemas conocidos: Core IDE

Éstos son los problemas no resueltos de esta versión:

  • Dificultades iniciar el IDE (Solaris x86 y Opteron Dual Core).

    Descripción: en algunas configuraciones con Solaris x86 y procesador Opteron Dual Core, pueden surgir problemas al iniciar el IDE debido a un error del JDK con la carga de bibliotecas GTK look&feel.

    Solución: si experimenta este problema, especifique la siguiente opción se encuentra en esta conducta, especifique la siguiente opción de línea de comandos: -laf javax.swing.plaf.metálica.MetalLookAndFeel

  • Los caracteres no aparecen correctamente en algunas configuraciones regionales asiáticas.

    Descripción: al ejecutar el IDE con algunas configuraciones regionales asiáticas en Ubuntu y en otras distribuciones de Linux, puede que el texto de la pantalla del IDE resulte ilegible. En determinados distribuciones de Linux, los caracteres multibyte pueden aparecer como rectángulos debido a problemas con la configuración de fuentes del JDK.

    Solución: consulte las FAQ para obtener ayuda sobre la solución del problema: http://wiki.netbeans.org/FaqI18nLinuxAsianFonts

  • Problema nº 6720021: surgen dificultades al iniciar el IDE en OpenSolaris con cualquier versión de JDK 5.

    Descripción: IDE puede no iniciarse en OpenSolaris al utilizar cualquier versión de JDK 5.

    Solución: utilice JDK 6 o ejecute el comando export AWT_TOOLKIT=XToolkit antes de iniciar el IDE.

  • Problema nº 64840: la comunicación de red se interrumpe en Linux si IPv6 está habilitado.

    Descripción: la comunicación de red se interrumpe en Linux si IPv6 está habilitado, lo que impide que el IDE se conecte a servidores agrupados, a las actualizaciones automáticas, etc.

    Solución: si surge este problema, ejecute el IDE con la instrucción switch -J-Djava.net.preferIPv4Stack=true'. También puede deshabilitar IPv6 (en /etc/módulos.conf o /etc/modprobe.conf).

    Para obtener más información consulte: Problema nº 5087907.

  • Problema nº 167401: el IDE se bloquea de forma aleatoria en los sistemas MacOS X que utilizan JDK 1.6.0 actualización 13

    Descripción: el IDE se puede bloquear de forma aleatoria al navegar por la barra del menú principal debido al error nº 6993269 del JDK de Apple.

    Solución: cambie a una versión distinta de JDK o modifique las opciones de inicio predeterminadas del archivo netbeans.conf a -J-Dapple.laf.useScreenMenuBar=false.

  • Problema nº 110074: el editor se bloquea en Solaris de forma periódica.

    Descripción: al ejecutar el IDE en Solaris con JDK 6, 6u1 o 6u2, los subprocesos pueden bloquearse durante mucho tiempo y provocar que el editor no responda.

    Solución: actualice la instalación de JDK 6 a la actualización 3 o más reciente.

  • Problema nº 88776: la aplicación perfilada puede fallar al crear perfiles con el modo de adjuntar de forma dinámica.

    Descripción: al ejecutarse en JDK 1.6, el creación del perfil de una aplicación con el modo de adjuntar de forma dinámica puede provocar un fallo de la aplicación perfilada.

    Solución: inicie la aplicación que desea perfilar con la instrucción de switch -Xshare:off.

    Consulte también: Problema nº 113847.

  • Problema nº 119617: el cursor se pierde en el editor y no se puede escribir con el teclado

    Descripción: algunas veces, tras usar una sugerencia de la ventana del editor, puede ocurrir que el punto de inserción del editor desaparezca y no pueda escribir con el teclado. Este problema ha surgido sobre todo en Mac OS X 10.4, pero también han surgido casos en algunas versiones de Ubuntu.

    Solución: utilice el ratón para abrir y cerrar una ventana de cuadro de diálogo (por ejemplo, un cuadro de diálogo de propiedades). De esta forma el editor vuelve a aparecer correctamente, y el funcionamiento del punto de inserción y del teclado vuelve a ser normal.

  • Problema nº 130357: I18N: Mac OS no genera javadoc si el nombre o la ruta del proyecto tiene varios bytes.

    Descripción: la tarea javadoc durante la creación del archivo de parámetros temporales se escribe en la codificación predeterminada del sistema operativo (MacRoman). Parece que se trata de una situación propia de Mac OS en la que las partes BSD y Mach (llamadas al sistema y al núcleo) aceptan parámetros en UTF-8, pero el sistema operativo y HFS se promocionan como MacRoman.

    Solución: evite el uso de varios bytes en el nombre o en la ruta del proyecto.

  • Problema nº 152916: surgen dificultades al iniciar IDE en Mac OS X 10.5.5 de 64 bits mediante JDK 1.6_07

    Descripción: en determinadas configuraciones de Mac OS X 10.5.5 de 64 bits con JDK 1.6_07 definido como plataforma Java predeterminada del sistema pueden surgir problemas al iniciar el IDE desde el Finder y desde la línea de comandos.

    Solución: cambie la plataforma predeterminada del sistema a JDK 1.5.

  • Problema nº 150416: la base de datos de muestra no se puede crear sin InnoDB

    Descripción: la base de datos de muestra de NetBeans emplea claves externas. El motor MyISAM no admite este tipo de claves. Por tanto, no se pueden utilizar bases de datos de muestra sin el motor de almacenamiento InnoDB.

    Solución: instale el motor de almacenamiento transaccional InnoDB.

  • Problema nº 158421: surgen dificultades al utilizar el IDE con el recolector de residuos G1

    Descripción: si utiliza el recolector de residuos G1 (en JDK 6 actualización 14 o posterior y en JDK 7), puede que se produzcan excepciones durante el uso del IDE.

    Solución: no utilice el recolector de residuos G1 (elimine -XX:+UnlockExperimentalVMOptions -XX:+UseG1GC desde netbeans.conf o desde la línea de comandos).

  • Problema nº 163371: no se pueden instalar aplicaciones web de Maven salvo que se seleccione un servidor de aplicaciones

    Solución: actualice manualmente RubyGems desde la línea de comandos: nb_install_dir_ruby2/jruby-1.4/bin/jruby -S gem update --system

Problemas conocidos: C/C++

NetBeans C/C++ es compatible con aplicaciones C y C++ en Microsoft Windows, Macintosh OS X, Linux y el sistema operativo SolarisTM. Los desarrolladores de C/C++ pueden integrar aplicaciones existentes en el sistema del proyecto de NetBeans o desarrollar aplicaciones y bibliotecas nuevas. La compatibilidad de NetBeans C/C++ abarca todo el ciclo de desarrollo: proyectos y plantillas, un avanzado editor compatible con el lenguaje, versiones basadas en la creación de archivos y depuración.

Para obtener información sobre la instalación y configuración de los compiladores y herramientas de compatibilidad con NetBeans C/C++, consulte la sección sobre instalación y configuración de la compatibilidad con C/C++.

Éstos son los problemas no resueltos de C/C++ de esta versión:

  • Problema nº 149053: en una plataforma Windows, la ruta de acceso a la creación de herramientas no puede contener espacios

    Descripción: en las plataformas Windows, si la ruta de acceso a la creación de herramientas contiene espacios integrados, se truncará y no será posible encontrar las herramientas.

    Solución: no utilice espacios en dicha ruta.

  • Problema nº 166811: GDB falla en Open Solaris al iniciar la depuración de archivos de C++

    Descripción: al depurar proyectos de C++ en Open Solaris, GDB falla debido a un error de segmentación.

    Solución: elimine los paquetes gcc-432 como se describe en la solución de este error.

    • Problema nº 148204: en las plataformas Windows, los datos introducidos por el usuario no funcionan en la ventana de salida al depurar

    Descripción: en las plataformas Windows, si se depura una aplicación y se ha definido la ventana de salida como tipo de consola del proyecto, no se puede introducir ningún dato en dicha ventana. Este problema no afecta al desarrollo mediante un host remoto.

    Solución: defina Terminal externa como tipo de consola; para ello, haga clic con el botón derecho del ratón en el nodo del proyecto de la ventana de proyectos, seleccione el nodo de ejecución en el cuadro de diálogo Propiedades del proyecto, seleccione Terminal externa en la lista desplegable Tipo de consola y haga clic en Aceptar.

  • Problema nº 121776: el código compilado de Sun Studio presenta problemas ocasionales al depurarlo con gdb

    Descripción: en los sistemas operativos Solaris y Linux, no siempre el posible depurar correctamente con el depurador gdb el código compilado con compiladores de Sun Studio. La mayoría de problemas conocidos se solucionan con el software Sun Studio 12, aunque existen varios más. Si experimenta problemas al depurar programas compilados con compiladores de Sun Studio 12, lo primero que debe hacer es aplicar los últimos parches de este software. Si el problema persiste, registre la incidencia en Issuezilla como defecto y haga que la tarea dependa de ella.

    Cuando surgen problemas, abrimos nuevas incidencias y actualizamos la tarea para que dependa de ellas, lo que ofrece una cierta perspectiva del estado actual de los programas de depuración compilados con compiladores de Sun Studio 12 mediante NetBeans y el depurador gdb.

    Estos problemas no afectan al código de depuración compilado con compiladores de Sun Studio 12 mediante Sun Studio 12 IDE y el depurador dbx.

  • Problema nº 121775: se necesita el Xcode más reciente para depurar en Tiger (Mac OS X 10.4)

    Descripción: al depurar en un equipo Macintosh con Tiger (Mac OS X 10.4), la versión predeterminada de los compiladores gcc y g++ genera código con un error en la salida de depuración. Apple ha lanzado un paquete de Xcode actualizado (versión 2.5) en el que este problema está solucionado. Las versiones correctas de gcc, g++ y gdb son: gcc: 4.0.1, compilación 5367 de Apple; g++: 4.0.1, compilación 5367 de Apple; gdb 6.3.50, compilación 573 de Apple (10/20/2006)

    Para descargar esta versión de Xcode, consulte http://developer.apple.com/tools/download.

    Este problema no aparece en Leopard (Mac OS X 10.5).

    Los síntomas de este problema son que los ejecutables compilados con el indicador -gdwarf (la predeterminada en proyectos de aplicaciones de C/C++) tienen información de líneas incorrecta y pueden ignorarse puntos de interrupción.
