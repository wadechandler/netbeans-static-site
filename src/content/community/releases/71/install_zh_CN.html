NetBeans IDE 7.1 安装说明

最新更新日期： October 31, 2013

本文档介绍了如何在系统上安装 NetBeans IDE 7.1。请参见 NetBeans IDE 7.1 发行说明，了解有关该 IDE 支持的操作系统和硬件配置信息。要了解此 IDE 发行版中所包含的新增功能，请参见 NetBeans IDE 7.1 发行信息页。

所需的软件

需要安装 Java SE Development Kit (JDK) 6 Update 26 或更高版本才能安装 NetBeans IDE。使用 JDK 5.0 无法安装 7.1 版本的 IDE。您可以在 http://www.oracle.com/technetwork/java/javase/downloads 中下载最新的 JDK 6 更新。

PHP 和 C/C++ NetBeans 包仅要求安装并运行 Java Runtime Environment (JRE) 6。不过，如果您打算使用所有的 Java 功能，则需要安装 JDK 6 或 JDK 7。

需要 JavaFX 2.0 SDK 才能在 NetBeans IDE 7.1 中使用 JavaFX 2.0 功能。您可以从 http://javafx.com/downloads/ 下载 JavaFX 2.0 SDK。

安装程序下载选项

NetBeans IDE 下载页上提供了多种安装程序，您可以下载其中的任何一种，每种安装程序都包含基本 IDE 和一些附加工具。

以下是可供选择的安装程序下载选项：

  • Java SE。支持所有标准的 Java SE 开发功能以及 NetBeans RCP 开发平台和 JavaFX 2.0 SDK。
  • Java EE。提供了用于开发 Java SE 和 Java EE 应用程序的工具，并且支持 NetBeans RCP 开发平台和 JavaFX 2.0 SDK。此下载选项还包括 GlassFish Server Open Source Edition 3.1.1 和 Apache Tomcat 7.0.22 软件。
  • C/C++。支持使用 C、C++、Qt、Fortran 和 Assembly 语言进行开发。
  • PHP。提供了 PHP 5.x 开发工具，并且支持 Zend 和 Symfony 框架。
  • All。此选项可以下载完整的 NetBeans IDE，其中包括该 IDE 的所有可用运行时环境和技术。

定制安装

如果使用 "All" 下载选项安装，则可以明确地选择要安装的工具和运行时环境。请在安装程序的欢迎页中单击“定制”并选择要使用的功能。有关详细信息，请参见安装软件。

如果要在 NetBeans IDE 配置中添加功能，请使用 NetBeans 插件管理器。例如，假定您先下载并安装了 PHP 包。随后，您决定要试用 C/C++ 功能。要执行此操作，请从 IDE 转至 NetBeans 插件管理器（选择“工具”>“插件”），然后将 C/C++ 包添加到现有安装中。

以下是有关安装和配置某些 NetBeans IDE 功能的几项重要说明：

  • "Java EE" 和 "All" 下载选项中均包含 Apache Tomcat 7.0.22，但这两个选项在缺省情况下都不安装该软件。要安装 Apache Tomcat，请选择下面的相应选项：
    • 如果通过 "Java EE" 下载选项进行安装，请启动安装程序，然后在“欢迎”页选中 "Apache Tomcat 7.0.22" 复选框；
    • 如果通过 "All" 下载选项进行安装，请启动安装程序，然后在“定制安装”对话框中选中 "Apache Tomcat 7.0.22"。
  • 对于 C/C++ 开发（适用于 "C/C++" 和 "All" 下载选项），您需要另外安装一些编译器和工具。有关详细信息，请参阅配置 NetBeans IDE 以使用 C/C++/Fortran 文档。
  • 要在 NetBeans IDE 7.1 发行版本中启用 JavaFX 2.0 支持，请参阅带 JavaFX 2.0 的 NetBeans IDE 文档。
  • 对于 Mac OS X，可以通过 "All" 下载选项定制 NetBeans IDE 安装。通过 "Java SE"、"Java EE"、"C/C++" 和 "PHP" 下载选项进行的 IDE 安装无法进行定制。
  • 您还可以下载与平台无关的 zip 文件，然后运行 NetBeans IDE 可执行文件。请注意，GlassFish Server Open Source Edition 和 Apache Tomcat 仅包含在特定于平台的安装程序中。
  • 要使样例项目模板在 NetBeans IDE 安装中无法访问，请参见隐藏样例项目常见问题解答页。

开始下载

  1. 转至 http://download.netbeans.org/netbeans/7.1
  2. 从该页面右上区域的下拉列表中选择语言和平台。您还可以选择下载并使用与平台无关的 zip 文件。
  3. 单击要安装的下载选项的“下载”按钮。
  4. 将安装文件保存到系统中。

安装软件

请按照此部分中的说明在系统上安装该 IDE。这些安装说明适用所有支持的平台。有关支持的平台和系统要求列表，请参见发行说明。

Microsoft Windows、Solaris OS 和 Linux

安装软件：

  1. 下载完成后，运行安装程序。
    • 对于 Windows，安装程序可执行文件的扩展名为 .exe。双击安装文件便可运行它。
    • 对于 Solaris 和 Linux 平台，安装文件的扩展名为 .sh。对于这些平台，您需要使用以下命令来执行安装文件：chmod +x <installer-file-name>
  2. 如果下载了 "All" 包，则可以对安装进行定制。请执行安装向导的“欢迎”页上的以下步骤：
    1. 单击“定制”。
    2. 在“定制安装”对话框中进行选择。
    3. 单击“确定”。
  3. 在安装向导的欢迎页中，单击“下一步”。
  4. 在“许可证协议”页中，查看许可证协议，单击接受复选框，然后单击“下一步”。
  5. 在“JUnit 许可证协议”页，确定是否要安装 JUnit，单击相应选项，然后单击“下一步”。
  6. 在 NetBeans IDE 安装页中，执行以下操作：
    1. 接受缺省的 NetBeans IDE 安装目录，或者指定其他目录。

      注意：安装目录必须为空，并且在运行安装程序时使用的用户身份必须具有此目录的读/写权限。

    2. 接受要与 NetBeans IDE 一起使用的缺省 JDK 安装，或者从下拉列表中选择其他安装。如果安装向导找不到要与 NetBeans IDE 一起使用的兼容 JDK 安装，则表明未在缺省位置安装 JDK。在这种情况下，指定安装的 JDK 路径，然后单击“下一步”，或者取消当前安装。在安装所需的 JDK 版本后，您可以重新启动安装。
  7. 如果打开了 GlassFish Server Open Source Edition 3.1.1 安装页，请接受缺省安装目录，或者指定其他安装位置。
  8. 如果您要在 Apache Tomcat 安装页上安装 Apache Tomcat，请接受缺省安装目录，或者指定其他安装位置。单击“下一步”。
  9. 在“摘要”页中，验证要安装的组件列表是否正确，以及系统上是否有足够的空间可用于安装。
  10. 单击“安装”开始进行安装。
  11. 在“安装完成”页，根据需要提供匿名使用数据，然后单击“完成”。

注意：如果您遇到问题而无法成功完成软件的安装，请参见疑难解答，该部分针对影响安装过程的待解决问题进行了描述并提供了建议的解决方法。

Mac OS X

安装软件：

  1. 下载完成后，运行安装程序。安装文件的扩展名为 .dmg。
  2. 在打开的面板上，单击软件包图标。软件包的扩展名为 .mpkg。将启动安装向导。
  3. 在安装向导的欢迎页中，单击“继续”。
  4. 查看许可证协议，然后单击“继续”。单击弹出式窗口中的“接受”以接受许可证。
  5. 在“选择目标”页中，选择驱动器，然后单击“继续”。
  6. 如果下载了 "All" 包，则可以对安装进行定制。在安装向导的最后一个面板上，按面板左下角的“定制”按钮。将显示产品树。
  7. 选择要安装的产品。
  8. 输入系统管理员的名称和口令，然后单击“确定”开始进行安装。

与平台无关的软件包

运行软件：

  1. 下载完与平台无关的 ZIP 文件之后，将其解压缩到系统上的任意文件夹中。
  2. 运行位于 netbeans/bin 目录中的可执行文件。
  3. 接受许可证协议。
  4. 在“JUnit 许可证协议”页，确定是否要安装 JUnit，单击相应选项，然后单击“下一步”。此时将会启动 NetBeans IDE 安装。

注意：如果找不到兼容的 JDK 安装，您可能需要通过执行以下操作手动添加指向 JDK 安装目录的路径：

  1. 在文本编辑器中打开位于 netbeans/etc 目录中的 netbeans.conf 文件。
  2. 为 netbeans_jdkhome 选项输入兼容 JDK 安装的位置。Windows 中的缺省位置为 C:\Program Files\Java\jdk1.7.0。
  3. 保存 netbeans.conf 文件，然后运行 netbeans/bin 目录中的可执行文件。

注意：GlassFish Server Open Source Edition 和 Apache Tomcat 仅包含在特定于平台的安装程序中。

注意：要使用已安装在计算机上的 GlassFish Server Open Source Edition 和/或 Apache Tomcat，请在 NetBeans IDE 中，单击“工具”菜单中的“服务器”。此时将会显示“添加服务器实例”向导，该向导可帮助在 NetBeans IDE 中注册所需 Web 服务器的实例。

卸载软件

请按照此部分中的说明在系统上卸载该 IDE。这些卸载说明适用所有支持的平台。有关支持的平台列表，请参见发行说明。

Microsoft Windows

如有必要，您可以使用以下步骤卸载 NetBeans IDE、GlassFish Server Open Source Edition 3.1.1 和 Apache Tomcat。

卸载 IDE：

  1. 关闭 IDE。
  2. 从“控制面板”中选择“添加或删除程序”，然后选择 "NetBeans IDE 7.1" 并单击“删除”。
  3. 如果要将 GlassFish 和 Tomcat 与 IDE 一同卸载，请确保“摘要”页中列出了正确的版本信息并选中相应的复选框。
  4. 如果要将 NetBeans 用户目录连同 IDE 一起删除，请选中 NetBeans 用户目录复选框，然后单击“卸载”。
  5. 在“摘要”页中，单击“卸载”。
  6. 卸载完成后，单击“完成”。

卸载 GlassFish Server Open Source Edition 3.1.1：

  1. 关闭 IDE。
  2. 从“控制面板”中选择“添加或删除程序”，然后选择 "GlassFish Server Open Source Edition 3.1.1" 并单击“删除”。

卸载 Apache Tomcat：

  1. 关闭 IDE。
  2. 从“控制面板”中选择“添加或删除程序”，然后选择 "Apache Tomcat 7.0.22" 并单击“删除”。

Solaris OS、Linux

如有必要，您可以使用以下步骤卸载 NetBeans IDE、GlassFish Server Open Source Edition 3.1.1 和 Apache Tomcat。

卸载 IDE：

  1. 关闭 IDE。
  2. 在 IDE 安装目录中，运行 uninstall.sh 文件。
  3. 如果要将 GlassFish 和 Tomcat 与 IDE 一同卸载，请确保“摘要”页中列出了正确的版本信息并选中相应的复选框。
  4. 在“摘要”页中，单击“卸载”。
  5. 卸载完成后，单击“完成”。

卸载 GlassFish Server Open Source Edition 3.1.1：

  1. 关闭 IDE。
  2. 找到该应用服务器的安装目录。

    注意：缺省安装位置因安装类型和平台而有所不同。下表显示了各种平台的缺省应用服务器安装目录。

    平台 安装类型... GlassFish 3.1.1 安装
    Windows 具有管理权限的用户 %ProgramFiles%\glassfish-3.1.1
    Windows 用户 %USERPROFILE%\glassfish-3.1.1
    Solaris OS 超级用户 /opt/glassfish-3.1.1
    Solaris OS、Linux 用户 ~/glassfish-3.1.1
    Linux 超级用户 /usr/local/glassfish-3.1.1
  3. 在应用服务器的安装目录中，运行 uninstall.sh 文件。
  4. 在“摘要”页中，单击“卸载”。
  5. 卸载完成后，单击“完成”。

卸载 Apache Tomcat：

  1. 关闭 IDE。
  2. 找到 Apache Tomcat 安装目录。

    注意：缺省安装位置因安装类型和平台而有所不同。下表显示了各种平台的缺省应用服务器安装目录。

    平台 安装类型... Apache Tomcat 7.0.22 安装
    Windows 具有管理权限的用户 %ProgramFiles%\Apache Software Foundation\Apache Tomcat 7.0.22
    Windows 用户 %USERPROFILE%\Apache Software Foundation\Apache Tomcat 7.0.22
    Solaris OS 超级用户 /opt/apache-tomcat-7.0.22
    Solaris OS、Linux 用户 ~/apache-tomcat-7.0.22
    Linux 超级用户 /usr/local/apache-tomcat-7.0.22
  3. 在应用服务器的安装目录中，运行 uninstall.sh 文件。
  4. 在“摘要”页中，单击“卸载”。
  5. 卸载完成后，单击“完成”。

Mac OS X

卸载 IDE 和应用服务器：

  1. 关闭软件。
  2. 将程序文件夹或文件拖至“废纸篓”。请注意，IDE 和应用服务器的缺省位置为 /Applications/NetBeans/。

与平台无关的软件包

要卸载 IDE 文件，请按照本文档的清理不完整的安装部分中所述的步骤进行操作。

升级和多重安装支持

多重安装的 NetBeans IDE 5.x 和 6.x 可以与 NetBeans IDE 7.1 在同一个系统上共存。无需卸载早期的版本，即可安装或运行 NetBeans IDE 7.1。

如果已经安装了 NetBeans IDE，则在首次运行 NetBeans IDE 7.1 时可以选择是否从现有用户目录中导入用户设置。

注意：此时无法对 IDE 执行增量式版本升级。要升级到最新版本，您必须下载最新的安装程序。

疑难解答

关于安装程序工作目录

安装程序在其工作目录中存储有关安装会话的数据。例如，此类数据包括系统中已安装的组件列表、安装日志消息以及注册表数据。

对于 Windows，安装程序工作目录的缺省位置为 %USERPROFILE%\.nbi；对于其他平台，缺省位置为 ~/.nbi。日志子目录中包含安装日志文件。

清理不完整的安装

如果未执行完整的安装或者安装不成功，则可能需要在系统中清理以前的安装。

对于 Microsoft Windows、Solaris OS 和 Linux：

  1. 删除安装程序工作目录。
  2. 删除 IDE 和 GlassFish 应用服务器的安装目录。
  3. 在 Windows 平台上，从以下注册表主键中删除所有 nbi* 注册表条目：HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

对于 Mac OS X：

  • 将程序文件夹或文件拖至“废纸篓”。

安装前禁用某些软件

开始执行安装前，最好禁用防火墙、防病毒和监视应用程序。它们中的某些应用程序可能会阻止安装过程中正确配置 IDE 组件。

例如，如果系统上正在运行 Logitech 进程监视器应用程序 (LVPrcSrv.exe)，请在执行安装前将其关闭。
