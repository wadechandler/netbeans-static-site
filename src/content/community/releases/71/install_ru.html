Дата последнего обновления: 25.04.12

В этом документе описывается процедура установки среды IDE NetBeans 7.1.2. Для получения информации о новых функциях, вошедших в данный выпуск среды IDE, см. страницу Информация о среде IDE NetBeans 7.1.2.

Требуемое программное обеспечение

Для установки среды IDE NetBeans требуется пакет разработчика Java SE (JDK) 6, обновление 26 или выше. Версию 7.1.2 среды IDE NetBeans нельзя установить с помощью пакета разработчика Java SE (JDK) 5.0. Последнее обновление JDK 6 можно загрузить по адресу http://www.oracle.com/technetwork/java/javase/downloads.

Для установки и запуска пакетов Netbeans для PHP и C/C++ требуется только установка среды Java Runtime Environment (JRE) 6. Однако если планируется использовать любую из функций Java, потребуется JDK 6 или JDK 7.

Пакет SDK для JavaFX 2.1 необходим для использования функций JavaFX 2.1 в среде IDE NetBeans 7.1.2. Пакет SDK для JavaFX 2.1 можно загрузить по адресу http://www.oracle.com/technetwork/java/javafx/downloads/index.html.

Варианты загрузки установщика

На Станице загрузки среды IDE NetBeans можно загрузить одну из нескольких программ установки, каждая из которых содержит основную среду IDE и дополнительные средства.

Доступны следующие программы установки:

  • Java SE. Поддерживает все стандартные функции разработки Java SE, а также платформу разработки RCP NetBeans и пакет SDK JavaFX 2.1.
  • Java EE. Предоставляет средства для разработки приложений Java SE и Java EE, а также поддержку платформы разработки RCP NetBeans и пакета SDK JavaFX 2.1. В состав этого варианта загрузки также входит сервер GlassFish Open Source Edition 3.1.2 и программное обеспечение Apache Tomcat 7.0.22.
  • C/C++. Реализована поддержка разработки на языках C, C++, Qt, Фортран и на языке ассемблера.
  • PHP. Предоставляет средства для разработки PHP 5.x и поддержки платформ Zend и Symfony.
  • Все. Это вариант полной загрузки, содержащий все среды выполнения и технологии, доступные для среды IDE NetBeans.

Индивидуализация установки

При выборе варианта загрузки Все можно указать конкретные средства и среды выполнения, которые требуется установить. Щелкните ссылку "Настроить" на странице "Добро пожаловать" и выберите требуемые компоненты. Дополнительные сведения приведены в разделе Установка программного обеспечения.

Для добавления функциональных возможностей к среде IDE NetBeans используется диспетчер подключаемых модулей NetBeans. Например, предположим, что сначала был загружен и установлен пакет PHP. После этого было принято решение опробовать функциональность C/C++. Для этого в среде IDE перейдите к диспетчеру подключаемых модулей NetBeans (выберите "Сервис > Подключаемые модули") и добавьте к существующей установке пакет "C/C++".

Ниже приводится ряд важных примечаний относительно установки и настройки некоторых компонентов среды IDE NetBeans:

  • Пакет Apache Tomcat 7.0.22 включен в варианты загрузки Java EE и Все, но по умолчанию не устанавливается ни в одном из этих вариантов. Для установки Apache Tomcat выберите подходящий вариант ниже:
    • в загрузке Java EE запустите программу установки и установите флажок для Apache Tomcat 7.0.22 на странице "Добро пожаловать";
    • в варианте загрузки Все запустите программу установки и выберите Apache Tomcat 7.0.22 в диалоговом окне "Изменить устанавливаемые компоненты".
  • Для разработки на C/C++ (применимо для вариантов загрузки C/C++ и Все) необходимо установить компиляторы и средства. Дополнительные сведения приведены в документе Настройка среды IDE NetBeans для языков C/C++/Фортран.
  • Сведения по включению поддержки JavaFX 2.1 в среде IDE NetBeans 7.1.2 приведены в документе Настройка среды NetBeans IDE с JavaFX 2.
  • Для платформы Mac OS X можно настроить установку среды IDE NetBeans из варианта загрузки Все. Установка среды IDE из вариантов загрузки Java SE, Java EE, C/C++ и PHP не может настраиваться.
  • Также можно загрузить независимый от платформы файл ZIP и запустить исполняемый файл среды IDE NetBeans. Следует отметить, что GlassFish Server Open Source Edition и Apache Tomcat доступны только с программами установки для конкретных платформ.
  • Сведения о том, как сделать шаблоны демонстрационных проектов недостижимыми из установки среды IDE NetBeans, приведены на странице Часто задаваемые вопросы по скрытию демонстрационных проектов.
  • Если загрузить и установить исправление для среды NetBeans IDE 7.1, обновленная среда IDE будет помечена как NetBeans 7.1 (не 7.1.2). Для обновления как SDK Java ME 3.0, так и Glassfish 3.1.1 запустите программу установки среды IDE NetBeans IDE 7.1.2.

Начало загрузки

  1. Перейдите к http://download.netbeans.org/netbeans/7.1.2
  2. Выберите соответствующую платформу и язык из раскрывающегося списка в верхней правой части страницы. Также можно выбрать для загрузки и использования файл ZIP, независимый от платформы.
  3. Нажмите кнопку "Загрузить", соответствующую выбранному варианту загрузки для установки.
  4. Сохраните файл программы установки.

Установка программного обеспечения

Для установки среды IDE следуйте указаниям, приведенным в этом разделе. Данные указания по установке применимы ко всем поддерживаемым платформам. Список поддерживаемых платформ и системных требований приведен в заметках к выпуску.

ОС Microsoft Windows, Solaris и Linux

Для установки программного обеспечения выполните следующие действия:

  1. После завершения загрузки запустите программу установки.
    • Для Windows исполняемый файл программы установки имеет расширение .exe. Чтобы запустить файл программы установки, дважды щелкните его.
    • Для платформ Solaris и Linux файл программы установки имеет расширение .sh. Для этих платформ необходимо выполнить следующую команду для исполняемых файлов программы установки: chmod +x <имя-файла-установщика>
  2. Если был загружен пакет Все, можно настроить индивидуальную установку. Выполните следующие действия на странице "Добро пожаловать" мастера установки:
    1. Нажмите кнопку "Настроить".
    2. В диалоговом окне "Изменить устанавливаемые компоненты" выберите требуемые варианты.
    3. Нажмите кнопку "ОК".
  3. На странице "Добро пожаловать" мастера установки нажмите кнопку "Далее".
  4. На странице "Лицензионное соглашение" ознакомьтесь с лицензионным соглашением, установите флажок для принятия его условий и нажмите кнопку "Далее".
  5. На странице "Лицензионное соглашение" JUnit решите, нужно ли устанавливать JUnit, и щелкните соответствующий вариант, затем нажмите кнопку "Далее".
  6. На странице установки среды IDE NetBeans выполните следующие действия:
    1. Выберите каталог для установки среды IDE NetBeans по умолчанию или укажите другой каталог.

      Примечание. Каталог установки должен быть пустым, а профиль пользователя, в котором выполняется программа установки, должен предусматривать полномочия на чтение и запись для этого каталога.

    2. Выберите установку комплекта JDK для использования со средой IDE NetBeans по умолчанию или укажите другую установку из раскрывающегося списка. Если мастеру установки не удается обнаружить совместимую установку комплекта JDK для использования со средой IDE NetBeans, значит, комплект JDK не установлен в местоположении по умолчанию. В этом случае укажите путь к установленному пакету JDK и нажмите кнопку "Далее" или отмените текущую установку. После установки необходимой версии пакета JDK можно перезапустить установку.
  7. При появлении страницы установки GlassFish Server Open Source Edition 3.1.2 выберите каталог установки по умолчанию или укажите другое местоположение для установки.
  8. Если выполняется установка сервера приложений Apache Tomcat, выберите на его странице установки каталог установки по умолчанию или укажите другое местоположение для установки. Нажмите кнопку "Далее".
  9. На странице "Сводка" проверьте правильность списка компонентов, которые требуется установить, а также наличие достаточного пространства в системе для установки.
  10. Для запуска процесса установки нажмите кнопку "Установить".
  11. На странице "Установка завершена" предоставьте анонимные данные об использовании (если необходимо) и нажмите кнопку "Готово".

Примечание. При возникновении проблем во время установки пакета ознакомьтесь с разделом Устранение неполадок, в котором содержится описание соответствующих процедур и возможных обходов проблем, влияющих на процесс установки.

OS X

Для установки программного обеспечения выполните следующие действия:

  1. После завершения загрузки запустите программу установки. Файл программы установки имеет расширение .dmg.
  2. На открывшейся панели щелкните значок пакета. Пакет имеет расширение .mpkg. Запускается мастер установки.
  3. На странице "Добро пожаловать" мастера установки нажмите кнопку "Продолжить".
  4. Прочтите лицензионное соглашение и нажмите кнопку "Продолжить". Для принятия условий лицензионного соглашения нажмите кнопку "Принимаю" во всплывающем окне.
  5. На странице "Выберите место назначения" выберите диск и нажмите кнопку "Продолжить".
  6. Если был загружен пакет Все, можно настроить индивидуальную установку. В нижней левой части последней панели мастера установки нажмите кнопку "Настроить". Появится дерево программ.
  7. Выберите программы, которые требуется установить.
  8. Введите соответствующее имя администратора и пароль и нажмите кнопку "ОК" для запуска установки.

Независимый от платформ пакет

Выполнение программы:

  1. После загрузки независимого от платформы файла ZIP распакуйте его в любую папку на компьютере.
  2. Запустите исполняемый файл, расположенный в каталоге netbeans/bin.
  3. Примите условия лицензионного соглашения.
  4. На странице "Лицензионное соглашение" JUnit решите, нужно ли устанавливать JUnit, и щелкните соответствующий вариант, затем нажмите кнопку "Далее". Будет запущена установка среды IDE NetBeans.

Примечание. Если совместимый установленный комплект JDK не обнаруживается, может потребоваться вручную добавить путь к нему в папку установки JDK, выполнив следующие действия:.

  1. Откройте в текстовом редакторе файл netbeans.conf, расположенный в каталоге netbeans/etc.
  2. Введите местоположение совместимого установленного комплекта JDK в качестве значения параметра netbeans_jdkhome. В Windows расположение по умолчанию – C:\Program Files\Java\jdk1.7.0_03.
  3. Сохраните файл netbeans.conf и выполните исполняемый файл в каталоге netbeans/bin.

Примечание. Сервер GlassFish Open Source Edition и Apache Tomcat доступны только с программами установки для конкретных платформ.

Примечание. Для использования сервера GlassFish Open Source Edition и/или Apache Tomcat, уже установленных на компьютере, в среде IDE NetBeans щелкните Серверы в меню Сервис. Отобразится мастер добавления экземпляра сервера, который поможет зарегистрировать экземпляр необходимого веб-сервера в среде IDE NetBeans.

Удаление программного обеспечения

Для удаления среды IDE следуйте указаниям, приведенным в этом разделе. Данные указания по удалению применимы ко всем поддерживаемым платформам. Список поддерживаемых платформ приводится в заметках к выпуску.

Microsoft Windows

При необходимости среду IDE NetBeans, GlassFish Server Open Source Edition 3.1.2 и Apache Tomcat можно удалить путем выполнения следующей последовательности действий.

Для удаления среды IDE выполните следующие действия:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите среду IDE NetBeans 7.1.2 и нажмите кнопку "Удалить".
  3. Для удаления GlassFish и Tomcat вместе со средой IDE на странице "Сводка" проверьте правильность версий и соответствующих флажков.
  4. Установите флажок для пользовательского каталога NetBeans, если его следует удалить вместе со средой IDE, и нажмите кнопку "Удалить".
  5. На странице "Сводка" нажмите кнопку "Удалить".
  6. После завершения удаления нажмите кнопку "Готово".

Для удаления сервера GlassFish Open Source Edition 3.1.2 выполните следующие действия:

  1. Завершите работу среды IDE.
  2. На панели управления выберите пункт "Установка и удаление программ", после чего выберите сервер GlassFish Open Source Edition 3.1.2 и нажмите кнопку "Удалить".

Для удаления Apache Tomcat выполните следующие действия:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите Apache Tomcat 7.0.22 и нажмите кнопку "Удалить".

ОС Solaris, Linux

При необходимости среду IDE NetBeans, сервер GlassFish Open Source Edition 3.1.2 и Apache Tomcat можно удалить путем выполнения следующей последовательности действий.

Для удаления среды IDE выполните следующие действия:

  1. Завершите работу среды IDE.
  2. В каталоге установки среды IDE запустите файл uninstall.sh.
  3. Для удаления GlassFish и Tomcat вместе со средой IDE на странице "Сводка" проверьте правильность версий и установите соответствующие флажки.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

Для удаления GlassFish Server Open Source Edition 3.1.2 выполните следующие действия:

  1. Завершите работу среды IDE.
  2. Определите каталог установки сервера приложений.

  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

Для удаления Apache Tomcat выполните следующие действия:

  1. Завершите работу среды IDE.
  2. Определите каталог установки Apache Tomcat.

  3. В каталоге установки сервера приложений запустите файл uninstall.sh.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

OS X

Для удаления среды IDE и сервера приложений выполните следующие действия:

  1. Завершите работу программного обеспечения.
  2. Перетащите папки или файлы программы в корзину. Учтите, что местоположением для среды IDE и сервера приложений по умолчанию является /Applications/NetBeans/.

Поддержка обновлений и множественных установок

Одновременно со средой IDE NetBeans 7.1.2 в одной системе могут существовать множественные установки среды IDE NetBeans 5.x , 6.x и 7.х. Для установки или запуска среды IDE NetBeans 7.1.2 удаление предыдущих версий не требуется.

Если в системе уже установлена среда IDE NetBeans, то при первом запуске NetBeans IDE 7.1.2 можно выбрать, следует ли импортировать пользовательские настройки из существующего пользовательского каталога.

Примечание. На этом этапе невозможно выполнить последовательные обновления версий среды IDE. Для обновления до новейшей версии необходимо загрузить самую позднюю версию установщика.

Устранение проблем

О рабочем каталоге программы установки

Программа установки сохраняет данные о сеансах установки в рабочем каталоге. В состав этих данных входят список установленных в системе компонентов, сообщения файла регистрации установки и информация из реестра.

Местоположением рабочего каталога программы установки по умолчанию является %USERPROFILE%\.nbi для ОС Windows и ~/.nbi для других платформ. Файлы журналов установки содержатся в подкаталоге журналов.

Очистка после незавершенных установок

После незавершенных или неуспешных установок может потребоваться очистка системы.

Для ОС Microsoft Windows, Solaris и Linux выполните следующие действия:

  1. Удалите рабочий каталог программы установки.
  2. Удалите каталоги установки среды IDE и сервера приложений GlassFish.
  3. На платформе Windows удалите все записи nbi* из следующих разделов системного реестра: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Для OS X:

  • Перетащите папки или файлы программы в корзину.

Отключение программного обеспечения перед установкой

Перед началом установки рекомендуется отключить брандмауэр, антивирусное программное обеспечение и приложения для наблюдения. Некоторые из этих приложений могут стать причиной некорректной настройки компонентов среды IDE в течение процесса установки.

Например, если в системе выполняется приложение монитора процессов Logitech (LVPrcSrv.exe), перед установкой его рекомендуется отключить.
