Notas da versión do IDE NetBeans 6.9.1

Última actualización: July 11, 2013

O IDE NetBeans é un contorno de desenvolvemento integrado (IDE) modular e baseado en estándares escrito na linguaxe de programación Java. O proxecto NetBeans consta dun IDE completo de código aberto escrito na linguaxe de programación Java e dunha plataforma de aplicativos cliente ricos, que se usa como un marco de traballo xenérico para construír calquera tipo de aplicativos. Para máis información sobre como instalar o software no seu sistema, consulte as instrucións de instalación do IDE NetBeans 6.9.1. Para obter máis información sobre as novas características incluídas nesta versión do IDE consulte a páxina de información da versión do IDE NetBeans 6.9.1.

Que é novo na versión 6.9.1

O IDE NetBeans é unha actualización de NetBeans 6.7.1 que inclúe os seguintes cambios:

  • Dispoñibilidade do último IDE empaquetado con JavaFX SDK 1.3.1
  • Melloras no depurador de JavaFX
  • Arranxos moi importantes nos problemas de usabilidade no Composer e no editor de JavaFX
  • Arranxos de erros críticos feitos noutros módulos desde a última versión final 6.9

As seguintes actualizacións de 6.8 incluidas na versión 6.9 tamén están en 6.9.1:

  • JavaFX Composer para facilitar e visualizar o desenvolvemento JavaFX
  • Compatibilidade cos marcos de traballos Felix OSGI e Equinox na plataforma NetBeans
  • Compatibilidade coa biblioteca do marco de traballo Spring 3.0
  • Java CardTM 3 conectada
  • Compatibilidade coa beta de Rails 3.0
  • Compatibilidade co marco de traballo PHP Zend
  • Compatibilidade con JSR-299, Inxección de dependencias e contextos (CDI)
  • Outras melloras adicionais listanse na páxina Novidades e notoriedades do IDE NetBeans 6.9.

Nota: O JDK 5.0 (Java TM SE Development Kit), non é compatíbel para calquera plataforma para esta versión. Para instalar e executar o IDE NetBeans 6.9.1 debe ter instalado o JDK 6.0 no seu sistema.


Para máis información sobre esta versión, consulte a páxina de Información da versión 6.9.1 do IDE NetBeans.

Compatibilidade con versións anteriores

Despois de rematar a instalación, cando execute o IDE por primeira vez ten a opción de importar algunhas das configuracións usadas na instalación anterior. Se non se opta pola importación de preferencias desde unha instalación anterior, o IDE ábrese usando as preferencias predefinidas. Teña en conta que o IDE unicamente recoñece instalacións anteriores cando o cartafol do usuario reside na localización predefinida. Se non se recoñecen instalacións, use --userdir para especificar un cartafol de usuario. Se desexa importar as preferencias desde unha localización que o instalador non recoñece, faga o seguinte:

  1. Inicie o IDE e no menú principal vaia a Ferramentas > Opcións.
  2. Prema no botón Importar do diálogo de Opcións.
  3. No diálogo Seleccionar opcións para importar, prema no botón Examinar e diríxase ao cartafol de usuario que contén as preferencias que desexa importar. Para máis información sobre a localización userdir do IDE NetBeans , consulte a FAQ Que é userdir e onde pode encontralo?

Nota: Se edita as propiedades do proxecto no IDE NetBeans 6.9.1 para un proxecto que foi creado co IDE NetBeans 4.1 ou anterior a este, o proxecto non volverá a funcionar nas versións anteriores do IDE NetBeans.

Problemas coñecidos: Instalador

Os problemas do instalador sen resolver para esta versión son os seguintes:

  • Problema #93568: en Solaris e Linux a instalación bloquease cando se perde a conexión de rede.

    Descrición: en plataformas Solaris e Linux, se se perde a conexión de rede, a instalación bloquease cando o instalador busca un JDK compatíbel nos cartafoles de rede.

    Solución: restaure a conexión de rede e continúe coa instalación.

  • Problema #117172: aparece unha xanela en branco ao executar o instalador en Ubuntu.

    Descrición: O diálogo do instalador aparece en branco/gris cando se executa en Ubuntu con Beryl/Compiz/Compiz Fusion instalado.

    Solución: execute a orde export AWT_TOOLKIT=MToolkit antes de executar o instalador.

  • Problema #123300. En Fedora Core 8, o instalador deixa de mostrar a interface e volve a consola.

    Descrición: en Fedora Core 8, a interface desaparece e o sistema volve a consola.

    Solución: Consulte https://netbeans.org/servlets/ReadMsg?list=nbusers&msgNo=98765.

  • O asistente de instalación do IDE NetBeans 6.9.1 non se inicia correctamente (colgase). Ocorre despois de que o usuario deteña o proceso de instalación manualmente e o programa de instalación lle informe de que hai outra instancia do instalador en execución.

    Descrición: en Windows, se se usa Java 6 de forma predefinida e se almacenan arquivos zip de gran tamaño no escritorio, o asistente de instalación non se inicia correctamente (bloquease). Ocorre despois de que o usuario deteña o proceso de instalación manualmente e o programa de instalación lle informe de hai outra instancia do instalador en execución.

    Solución: antes da instalación, elimine todos os arquivos zip de gran tamaño do escritorio.

  • Problema #177872: En Mac OS X, o complemento de GlassFish 3.01, e as ferramentas de actualización non funcionan correctamente

    Descrición: En Mac OS X, os complementos baseados con pkg e as ferramentas de actualización non funcionan correctamente baixo contas que non sexan administrativas se GlassFish v3.01 foi instalado como usuario administrativo (root).

    Solución temporal: Use a orde chown -R para cambiar o propietario da instalación de GlassFish e estabelecela a un usuario non root que o necesite. Por exemplo: sudo chown -R <novousuario>:admin /Applications/NetBeans/glassfish-3.0.1, onde novousuario é o usuario non administrador. Despois de usar esta orde, o usuario pode executar directamente as ferramentas bin/updatetool e bin/pkg.

  • Para máis información sobre a resolución de problemas do Instalador, diríxase á FAQ de usuario da instalación do NetBeans.

Problemas coñecidos: Núcleo do IDE

Os problemas non solucionados para esta versión son os seguintes:

  • Dificultades ao iniciar o IDE (Solaris x86 e Dual-Core Opteron).

    Descrición: En certas configuracións de Solaris x86 e Dual Core Opteron, pode encontrar problemas ao iniciar o IDE debido ao erro do JDK ao cargar as bibliotecas de aparencia de GTK.

    Solución: Se encontra este comportamento, especifique a seguinte opción da liña de ordes -laf javax.swing.plaf.metal.MetalLookAndFeel

  • Nas configuracións rexionais Asiáticas non se mostran correctamente os caracteres.

    Descrición: Ao executar o IDE con algunhas configuracións rexionais asiáticas en Ubuntu e outras distribucións de Linux, o texto de pantalla do IDE pode resultar non lexíbel. En determinadas distribucións de Linux, os caracteres multibyte poden aparecer como rectángulos debido a problemas coa configuración de fontes do JDK.

    Solución: consulte as FAQ para obter axuda sobre a solución do problema: http://wiki.netbeans.org/FaqI18nLinuxAsianFonts

  • Problema #64840: a comunicación de rede interrompese en Linux se IPv6 está activado.

    Descrición: a comunicación de rede interrompese en Linux se IPv6 está activado, o que impide que o IDE se conecte a servidores agrupados, ás actualizacións automáticas, etc.

    Solución: se xorde este problema, execute o IDE coa instrución switch -J-Djava.net.preferIPv4Stack=true' Tamén pode desactivar IPv6 (en /etc/modules.conf ou /etc/modprobe.conf).

    Para obter máis información consulte: Problema #5087907.

  • Problema #167401: o IDE bloquease de forma aleatoria nos sistemas MacOS X que usan JDK 1.6.0 actualización 13

    Descrición: o IDE pódese bloquear de forma aleatoria ao navegar pola barra do menú principal debido ao erro #6993269 do JDK de Apple.

    Solución: Cambie a unha versión diferente do JDK, ou ben modifique as opcións predefinidas de inicio do ficheiro netbeans.conf a -J-Dapple.laf.useScreenMenuBar=false.

  • Problema #110074: o editor bloquease en Solaris de forma periódica.

    Descricion: ó executar o IDE en Solaris con JDK 6, 6u1 ou 6u2, os subprocesos poden bloquearse durante moito tempo e provocar que o editor non responda.

    Solución: actualice a instalación de JDK 6 á actualización 3 ou máis recente.

  • Problema #88776: o aplicativo perfilado pode fallar ao crear perfiles co modo de anexar de forma dinámica.

    Descrición: ó executarse no JDK 1.6, a creación do perfil dun aplicativo co modo de anexar de forma dinámica podo provocar un fallo do aplicativo perfilado.

    Solución: inicie o aplicativo que desexa perfilar coa instrución de switch -Xshare:off.

    Consulte tamén: problema #113847.

  • Problema #119617: o cursor pérdese no editor e non se pode escribir co teclado

    Descrición: algunhas veces, despois de usar unha suxestión da xanela do editor, pode ocorrer que o punto de inserción do editor desapareza e non poida escribir co teclado. Maioritariamente informouse deste problema en Mac OS X 10.4, pero tamén se informou desde algunha das versións de Ubuntu.

    Solución: Use o rato para abrir e pechar unha xanela do cadro de diálogo (por exemplo, un cadro de diálogo de propiedades). Desta forma o editor volve a aparecer correctamente, e o funcionamento do punto de inserción e do teclado volve a ser normal.

  • Problema #130357: I18N - Mac OS non xenerará javadoc se o nome do proxecto ou o seu camiño conteñen caracteres multibyte.

    Descrición: A tarefa de javadoc está escrita con MacRoman, a codificación predefinida en Mac OS, cando se crea o ficheiro de parámetros temporal. Isto pode producirse por partes de BSD ou de Mach (chamadas ao sistema e o núcleo) que aceptan parámetros en UTF-8, pero Mac OS e HFS unicamente aceptan MacRoman.

    Solución: Deixe de usar caracteres multibyte no nome ou no camiño do proxecto.

  • Problema #152916: xorden dificultades ao iniciar o IDE en Mac OS X 10.5.5 de 64 bits mediante JDK 1.6_07

    Descrición: en determinadas configuracións de Mac OS X 10.5.5 de 64 bits con JDK 1.6_07 definido como plataforma Java predefinida do sistema poden xurdir problemas ao iniciar o IDE desde o Finder e desde a liña de ordes.

    Solución: Actualice a unha versión máis recente do JDK 6.0.

  • Problema #158421: xorden dificultades ao usar o IDE co recolector de lixo G1

    Descrición: Se está usando un Colector de lixo G1 (no JDK 6 Update 14 ou superior e no JDK 7), podería encontrar excepcións ao usar o IDE.

    Solución temporal: Non use o Colector de Lixo G1 (elimine -XX:+UnlockExperimentalVMOptions -XX:+UseG1GC do netbeans.conf ou na liña de ordes).

  • Para obter máis información sobre a resolución de problemas do IDE diríxase aFAQ do usuario de NetBeans.

Problemas coñecidos: Web e Java EE

Estes son os problemas sen resolver de Web e de Java Web desta versión:

  • Problema #183614: Paso a paso, non funciona correctamente con EJB.

    Descrición: Cando se depura un servlet, paso a paso a unha chamada de EJB e use o paso a través.

    Solución: Poña un punto de interrupción na implementación do códigodo EJB e use paso a través.

  • Problema #178108: As clases Metamodel para entidades JPA xestionadas non se xeran de forma automática.

    Descrición: As clases Metamodel para as correspondentes entidades xestionadas non se xeran. Ao construír consultas de tipos seguros (typesafe) que usan a API Criteria e a clase metamodel, non funcionan.

    Solución temporal: Vexa a descrición do problema #178108 para obter as opcións como solución temporal.

  • Para máis informacón sobre a resolución de problemas no módulo Web e Java EE, diríxase á FAQ do usuario de NetBeans.

Problemas coñecidos: Java ME

NetBeans permítelle desenvolver aplicatrivos baseados en Java ME para dispositivos MIDP, CLDC e CDC.

Os problemas non resoltos no Java ME para esta versión son os seguintes:

  • Java ME SDK 3.0 non é compatíbel con Linux, Solaris e outros sistema operativos.

    Descrición: NetBeans 6.9.1 Mobility e o paquete Java ME SDK 3.0 nesta versión só é compatíbel cos sistemas operativos Windows e Macintosh.

    Solución: O instalador de LInux ven coa versión 2.5.2 de Sun Java Wireless Toolkit.

  • Non é posíbel executar o módulo de Java ME se un nome de cartafol, camiño ao cartafol de instalación ou o cartafol de usuario contén caracteres non ASCII.

    Descrición: O SDK de Java ME 3.0 non é compatíbel cos caracteres non ASCII nos camiños dos proxectos.

    Solución: Asegúrese de que o seu cartafol de usuario non conteña carecteres que non sexan ASCII. Neste caso, cambie o cartafol de usuario predefinido do IDE.

  • Problema #139031: o emulador non se inicia en Windows se o binario de Java non se encontra no camiño do entorno.

    Descrición: Java ME SDK e outros emuladores non se inician en Windows se Java.exe non se encontra no camiño.

    Solución: instale o JDK coa configuración predefinida ou, se vai a realizar unha instalación personalizada, asegúrese de seleccionar o JRE público para a instalación. Se a versión do JDK xa está instalada, pode agregar o ficheiro java.exe á configuración do camiño do sistema.

  • Para obter información adicional sobre a resolución de problemas con Java ME, diríxase á FAQ de usuarios de NetBeans sobre Java ME e Conexión de mobilidade.

Problemas coñecidos: C/C++

NetBeans C/C++ é compatíbel con aplicativos C e C++ en Microsoft Windows, Macintosh OS X, Linux e o sistema operativo SolarisTM. Os desenvolvedores de C/C++ poden integrar aplicativos existentes no sistema do proxecto de NetBeans ou desenvolver aplicativos e bibliotecas novas. A compatibilidade de NetBeans C/C++ abarca todo o ciclo de desenvolvemento, proxectos e modelos, un avanzado editor compatíbel coa linguaxe, versións baseadas na creación de ficheiros e depuración.

Para obter información sobre a instalación e configuración dos compiladores e ferramentas de compatibilidade con NetBeans C/C++, consulte a sección sobre a instalación e configuración da compatibilidade con C/C++.

Os problemas de C/C++ sen resolver para esta versión son os seguintes:

  • Problema #148204: nas plataformas Windows, os datos introducidos polo usuario non funcionan na xanela de saída ao depurar

    Descrición: nas plataformas Windows, se se depurar un aplicativo e se definiu a xanela de saída como tipo de consola do proxecto, non se pode introducir ningún dato en dita xanela. Este problema non afecta ao desenvolvemento mediante un equipo remoto.

    Solución: defina Terminal externa como tipo de consola; para elo, prema co botón dereito do rato no nodo do proxecto da xanela de proxectos, seleccione o nodo de execución no cadro de diálogo Propiedades do proxecto, seleccione Terminal externa na lista despregábel Tipo de consola e prema en Aceptar.

  • Problema #121775: requírese a última versión de Xcode para a depuración en Tiger (Mac OS X 10.4)

    Descrición: Cando se depura en Mac OS Tiger (Mac OS X 10.4), o código xerado pola versión predefinida dos compiladores de gcc e de g++ teñen unha liña de información incorrecta e os puntos de interrupción poden ser ignorados.

    Solución:Apple lanzou unha actualización do paquete Xcode (versión 2.5) que corrixe este problema. As versións correctas de gcc, g++ e gdb son:

    • gcc: 4.0.1 - Apple build 5367
    • g++: 4.0.1 - Apple build 5367
    • gdb: 6.3.50 - Apple build 573 (10/20/2006)

    A versión actualizada de Xcode está dispoñíbel en: http://developer.apple.com/tools/download.

  • Para obter información adicional de resolución de problemas e asistencia para C/C++, diríxase a FAQ de C/C++ para NetBeans.
