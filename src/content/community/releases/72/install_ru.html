Дата последнего обновления: October 31, 2013

Требуемое программное обеспечение

Для установки среды IDE NetBeans требуется комплект разработчика Java (JDK) 6 Update 26 или более поздняя версия. Последнее обновление JDK 6 или 7 можно загрузить по адресу: http://www.oracle.com/technetwork/java/javase/downloads.

Примечание. Версию 7.2 IDE нельзя установить с помощью JDK 5.0.

Для установки пакетов PHP и C/C++ NetBeans требуется только установленная и функционирующая среда выполнения Java (JRE) 6. Однако, если необходимо использовать какие-либо функциональные возможности Java, потребуется JDK 6 или JDK 7.

Для использования функций JavaFX 2.1.1 в IDE NetBeans 7.2 требуется SDK JavaFX 2.1.1. SDK JavaFX 2.1.1 можно загрузить на сайте загрузок JavaFX.

Примечание. В зависимости от операционной системы, можно установить JavaFX 2.1.1 в комплекте с JDK Java. В противном случае требуется отдельная установка JavaFX 2.1.1.

  • Для ОС Windows JavaFX 2.1.1 предоставляется в наборе с JDK 1.7 Update 4. При установке входящего в комплект Java FX 2.1.1 (библиотека времени исполнения и пакет SDK) в местоположения по умолчанию IDE NetBeans может автоматически создать платформу Java с поддержкой JavaFX.
  • Для OS X JavaFX 2.1.1 встроена в Mac JDK 1.7 Update 4. IDE NetBeans может автоматически создать платформу Java с поддержкой JavaFX.
  • JavaFX работает не во всех операционных системах. Дополнительные сведения о версиях платформ с поддержкой JavaFX см. в документе Системные требования для JavaFX 2.1.1. Системные требования для JavaFX 2.1.1 также позволяют установить IDE NetBeans.

В начало

Варианты загрузки программы установки

На странице загрузок IDE NetBeans можно получить одну из нескольких программ установки, каждая из которых содержит базовую среду IDE и дополнительные средства.

Доступны следующие программы установки:

  • Java SE. Поддерживаются все стандартные функции разработки Java SE, а также поддерживается разработка на платформе NetBeans и SDK JavaFX 2.1.1.
  • Java EE. Предоставляет средства для разработки приложений Java SE и Java EE, а также поддержку разработки платформы NetBeans и SDK JavaFX 2.1.1. В состав данного варианта загрузки входит также сервер GlassFish Server Open Source Edition 3.1.2.2 и программный пакет Apache Tomcat 7.0.27.
  • C/C++. Реализована поддержка разработки на языках C, C++, Qt, Фортран и на языке ассемблера.
  • PHP. Предоставляет средства для разработки PHP 5.x и поддержки платформ Zend и Symfony.
  • Все. Это загрузка полного пакета, содержащая все библиотеки времени исполнения и технологии, доступные для IDE NetBeans.

В начало

Индивидуализация процесса установки

Вариант загрузкиВсе позволяет выбрать средства и среды выполнения для установки. Щелкните ссылку "Настроить" на странице "Добро пожаловать" и выберите требуемые компоненты. Дополнительные сведения см. в разделе Установка программного обеспечения.

Если необходимо добавить функциональность к конфигурации IDE NetBeans, используйте диспетчер подключаемых модулей NetBeans. Допустим, например, что вы начинаете с загрузки и установки пакета PHP. Затем вы приняли решение испытать функциональные возможности C/C++. Для этого в среде IDE перейдите к диспетчеру подключаемых модулей NetBeans (выберите "Сервис > Подключаемые модули") и добавьте к существующей установке пакет "C/C++".

Существует несколько важных примечаний к установке и конфигурации некоторых функций IDE NetBeans:

  • Пакет Apache Tomcat 7.0.27 включен в варианты загрузки Java EE и Все, но не устанавливается по умолчанию ни в одном из этих вариантов. Для установки Apache Tomcat выберите нужный вариант из перечисленных ниже:
    • из загрузки Java EE запустите программу установки и установите флажок Apache Tomcat 7.0.27 на странице приветствия;
    • из загрузки Все запустите программу установки и выберите Apache Tomcat 7.0.27 в диалоговом окне "Настроить устанавливаемые компоненты".
  • Чтобы включить поддержку JavaFX 2.1.1 в версии IDE NetBeans 7.2, см. документ Настройка IDE NetBeans с использованием JavaFX 2.1.
  • Для OS X можно настроить установку IDE NetBeans с помощью загрузок Все и Java EE. В случае установки среды IDE из загрузок Java SE, C/C++ и PHP индивидуальная настройка невозможна.
  • Также можно загрузить файл в формате zip, не зависящий от платформы, и запустить исполняемый файл IDE NetBeans. Обратите внимание, что GlassFish Server Open Source Edition и Apache Tomcat доступны только в составе программ установки для определенных платформ.
  • Чтобы сделать шаблоны образцов проектов в установке IDE NetBeans недоступными, см. страницу Часто задаваемые вопросы по скрытию образцов проектов.

В начало

Начало загрузки

  1. Перейдите по ссылке http://download.netbeans.org/netbeans/7.2/
  2. Выберите соответствующую платформу и язык из выпадающего списка в верхней правой части страницы. Также для загрузки и использования можно выбрать zip-файл, не зависящий от платформы.
  3. Нажмите кнопку "Загрузить", соответствующую выбранному варианту загрузки.
  4. Сохраните файл программы установки на вашем компьютере.

В начало

Установка программного обеспечения

Для установки среды IDE следуйте указаниям, приведенным в этом разделе. Данные инструкции по установке применяются ко всем поддерживаемым платформам. Список поддерживаемых платформ и системных требований приведен в заметках о выпуске.

ОС Microsoft Windows, Solaris и Linux

Для установки программного обеспечения:

  1. После завершения загрузки запустите программу установки.
    • Для Windows исполняемый файл программы установки имеет расширение .exe. Чтобы запустить файл программы установки, дважды щелкните его.
    • Для платформ Solaris и Linux файл программы установки имеет расширение .sh. На этих платформах необходимо выполнить следующую команду для исполняемых файлов программы установки: chmod +x <installer-file-name>
  2. В случае загрузки пакета Все или Java EE возможна пользовательская настройка установки. Выполните следующие действия на странице приветствия мастера установки:
    1. Нажмите кнопку "Настроить".
    2. В диалоговом окне "Изменить устанавливаемые компоненты" выберите нужные варианты.
    3. Нажмите кнопку "ОК".
  3. На странице "Добро пожаловать" мастера установки нажмите кнопку "Далее".
  4. На странице "Лицензионное соглашение" ознакомьтесь с лицензионным соглашением, установите флажок принятия условий и нажмите кнопку "Далее".
  5. На странице "Лицензионное соглашение JUnit" определите необходимость установки JUnit, щелкните соответствующий параметр и нажмите кнопку "Далее".
  6. На странице установки IDE NetBeans выполните следующие действия:
    1. Оставьте без изменения каталог установки по умолчанию для IDE NetBeans или укажите другой каталог.

      Примечание. Каталог установки должен быть пустым, а используемый профиль пользователя, чтобы запустить установщик, должен иметь разрешения на чтение / запись для этого каталога.

    2. Оставьте без изменения установку JDK для использования с IDE NetBeans или выберите в раскрываемом списке другую установку. Если мастеру установки не удалось найти совместимую установку JDK для использования с IDE NetBeans, JDK не будет установлен в местоположение по умолчанию. В этом случае укажите путь к установленному пакету JDK и нажмите кнопку "Далее" или отмените текущую установку. После установки необходимой версии пакета JDK можно перезапустить установку.
  7. При появлении страницы установки GlassFish Server Open Source Edition 3.1.2.2 выберите каталог установки по умолчанию или укажите иное местоположение для установки.
  8. Если выполняется установка сервера приложений Apache Tomcat, выберите на его странице установки каталог установки по умолчанию или укажите другое местоположение для установки. Нажмите кнопку "Далее".
  9. На странице "Сводка" проверьте правильность списка компонентов, которые требуется установить, а также наличие в системе требуемого свободного пространства для установки.
  10. Для запуска процесса установки нажмите кнопку "Установить".
  11. На странице "Установка завершена" по желанию укажите анонимные данные об использовании и нажмите кнопку "Далее".

Примечание. Если вы столкнулись с ошибками при установке программного обеспечения см. раздел Диагностика для доступа к описанию и временных решений проблем, которые могут повлиять на процесс установки.

OS X

Для установки программного обеспечения:

  1. После завершения загрузки запустите программу установки. Файл программы установки имеет расширение .dmg.
  2. На открывшейся панели дважды щелкните значок пакета. Пакет имеет расширение .mpkg. Запускается мастер установки.
  3. В отобразившемся диалоговом окне "Пакет выполнит запуск программы для проверки возможности установки программного обеспечения" нажмите кнопку "Продолжить".
  4. На странице "Введение" мастера установки нажмите кнопку "Продолжить".
  5. Прочтите лицензионное соглашение и нажмите кнопку "Продолжить". Для принятия условий лицензионного соглашения нажмите кнопку "Принимаю" во всплывающем окне.
  6. На странице "Выбор назначения" выберите диск и нажмите кнопку "Продолжить".
  7. В случае загрузки пакета Все или Java EE возможна пользовательская настройка установки. В нижней левой части последней панели мастера установки нажмите кнопку "Настроить". Отображается дерево программ.
  8. Выберите программы, которые требуется установить.
  9. Введите имя администратора и пароль и нажмите кнопку "ОК" для запуска установки.

Пакет, не зависящий от платформы

Для запуска программного обеспечения:

  1. После загрузки не зависящего от платформы zip-файла распакуйте его в любую папку на вашем компьютере.
  2. Запустите исполняемый файл, расположенный в каталоге netbeans/bin.
  3. Примите условия лицензионного соглашения.
  4. На странице "Лицензионное соглашение JUnit" определите необходимость установки JUnit, щелкните соответствующий параметр и нажмите кнопку "Далее". Будет начата установка IDE NetBeans.

Примечание. Если совместимый установленный комплект JDK не обнаружен, может потребоваться вручную добавить путь к этому пакету в папку установки JDK. Выполните следующие действия:

  1. Откройте в текстовом редакторе файл netbeans.conf, расположенный в каталоге netbeans/etc.
  2. Укажите местоположение совместимого установленного комплекта JDK в качестве значения параметра netbeans_jdkhome. Местоположение по умолчанию в Windows — C:\Program Files\Java\jdk1.7.0_03.
  3. Сохраните файл netbeans.conf и запустите исполняемый файл в каталоге netbeans/bin.

Примечание. GlassFish Server Open Source Edition и Apache Tomcat доступны только в составе программ установки для определенных платформ.

Примечание. Для использования сервера GlassFish Open Source Edition и/или программного пакета Apache Tomcat, которые уже установлены на компьютере пользователя, в IDE NetBeans щелкните Серверы в меню Сервис. Отобразится мастер добавления экземпляров серверов, используемый для регистрации экземпляра необходимо веб-сервера в IDE NetBeans.

В начало

Удаление программного обеспечения

Для удаления среды IDE следуйте инструкциям, приведенным в этом разделе. Данные инструкции по удалению применяются ко всем поддерживаемым платформам. Список поддерживаемых платформ приведен в заметках о выпуске.

Microsoft Windows

При необходимости можно удалить IDE NetBeans, сервер GlassFish Open Source Edition 3.1.2.2 и программный пакет Apache Tomcat с помощью следующих процедур.

Для удаления среды IDE:

  1. Завершите работу среды IDE.
  2. На панели управления выберите 'Установка и удаление программ', а затем выберите IDE NetBeans 7.2 и щелкните 'Удалить'.
  3. Для удаления компонентов GlassFish и Tomcat вместе со средой IDE на странице "Сводка" проверьте правильность версий и установите соответствующие флажки.
  4. Установите флажок для каталога пользователя NetBeans, если требуется удалить этот каталог вместе со средой IDE, и нажмите "Удалить".
  5. На странице "Сводка" нажмите кнопку "Удалить".
  6. После завершения удаления нажмите кнопку "Готово".

Удаление компонентов GlassFish Server Open Source Edition 3.1.2.2:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите GlassFish Server Open Source Edition 3.1.2.2 и нажмите "Удалить".

Для удаления компонентов Apache Tomcat:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите Apache Tomcat 7.0.27 и нажмите "Удалить".

ОС Solaris и Linux

При необходимости можно удалить IDE NetBeans, сервер GlassFish Open Source Edition 3.1.2.2 и программный пакет Apache Tomcat с помощью следующих процедур.

Удаление среды IDE:

  1. Завершите работу среды IDE.
  2. В каталоге установки IDE запустите файл uninstall.sh.
  3. Для удаления GlassFish и Tomcat вместе со средой IDE на странице "Сводка" проверьте правильность версий и установите соответствующие флажки.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

Удаление компонентов GlassFish Server Open Source Edition 3.1.2.2:

  1. Завершите работу среды IDE.
  2. Укажите каталог установки сервера приложений.

    Примечание. Местоположение установки по умолчанию зависит от типа установки и платформы. В следующей таблице приводятся каталоги установки сервера приложений по умолчанию для различных платформ.

    Платформа Установка от имени... Установка GlassFish 3.1.2.2
    Windows пользователь с правами администратора %ProgramFiles%\glassfish-3.1.2.2
    Windows пользователь %USERPROFILE%\glassfish-3.1.2.2
    ОС Solaris корень /opt/glassfish-3.1.2.2
    ОС Solaris, Linux пользователь ~/glassfish-3.1.2.2
    Linux корень /usr/local/glassfish-3.1.2.2
  3. В каталоге установки сервера приложений запустите файл uninstall.sh.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

Удаление пакета Apache Tomcat:

  1. Завершите работу среды IDE.
  2. Укажите каталог установки Apache Tomcat.

    Примечание. Местоположение установки по умолчанию зависит от типа установки и платформы. В следующей таблице приводятся каталоги установки сервера приложений по умолчанию для разных платформ.

    Платформа Установка от имени... Установка Apache Tomcat 7.0.27
    Windows пользователь с правами администратора %ProgramFiles%\Apache Software Foundation\Apache Tomcat 7.0.27
    Windows пользователь %USERPROFILE%\Apache Software Foundation\Apache Tomcat 7.0.27
    ОС Solaris корень /opt/apache-tomcat-7.0.27
    ОС Solaris, Linux пользователь ~/apache-tomcat-7.0.27
    Linux корень /usr/local/apache-tomcat-7.0.27
  3. В каталоге установки сервера приложений запустите файл uninstall.sh.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

OS X

Для удаления среды IDE и сервера приложений:

  1. Завершите работу программного обеспечения.
  2. Перетащите папки или файлы программы в корзину. Обратите внимание, что местоположением для среды IDE и сервера приложений по умолчанию является /Applications/NetBeans/.

Пакет, не зависящий от платформы

Для удаления файлов среды IDE выполните действия, описанные в разделе Очистка после незавершенных установок этого документа.

В начало

Поддержка обновлений и множественных установок

В одном системе допускается совместное использование нескольких установок IDE NetBeans 5.x, 6.x и 7.x с IDE NetBeans 7.2. Для установки или запуска IDE NetBeans 7.2 не требуется удалять предыдущие версии.

При наличии существующей установки IDE NetBeans можно определить, требуется ли импортировать настройки пользователей из существующего каталога пользователя при первом запуске IDE NetBeans 7.2.

Примечание. На данном этапе невозможно выполнить последовательные обновления версий среды IDE. Для обновления до новейшей версии необходимо загрузить последнюю версию программы установки.

Устранение проблем

О рабочем каталоге программы установки

Программа установки сохраняет данные о сеансах установки в рабочем каталоге. Эти данные включают список установленных в системе компонентов, сообщения журнала установки и данные из реестра.

Местоположением рабочего каталога программы установки по умолчанию является %USERPROFILE%\.nbi для ОС Windows и ~/.nbi для других платформ. Файлы журнала установки находятся в подкаталоге журнала.

Очистка после незавершенных установок

После незавершенных или неуспешных установок может потребоваться очистка системы.

Для ОС Microsoft Windows, Solaris и Linux:

  1. Удалите рабочий каталог программы установки.
  2. Удалите каталоги установки среды IDE и сервера приложений GlassFish.
  3. На платформе Windows следует удалить все записи вида nbi* из следующих ключей реестра: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall, HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Для OS X:

  • Перетащите папки или файлы программы в корзину.

Отключение некоторых программ до установки

Перед началом установки рекомендуется отключить брандмауэр, антивирусное программное обеспечение и приложения мониторинга. Некоторые из этих приложений могут стать причиной некорректной настройки компонентов среды IDE в процессе установки.

Например, если в системе работает приложение мониторинга процессов Logitech (LVPrcSrv.exe), перед установкой его рекомендуется отключить.
